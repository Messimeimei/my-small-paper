# 1. RESEARCH CONTEXT POSITIONING

**Independent Positioning within the Landscape:**

The submission addresses the problem of generating *interpretable* topic titles and descriptions from large-scale, unstructured user-contributed comments (YouTube). It proposes a method (TDRG) leveraging LLMs (Llama-3.1-8B-Instruct, fine-tuned with LoRA) for zero-shot and fine-tuned generation, evaluated via semantic similarity to human references.

**Most Closely Related Prior Works:**
- **Topic-driven Reader Comments Summarization (MSTM/EXTM, Ma et al. 2012):** Uses probabilistic topic models to cluster and summarize comments, but does not generate explicit topic titles/descriptions in natural language.
- **TWAG (Topic-Guided Wikipedia Abstract Generator):** Generates Wikipedia abstracts with topic guidance, but not from user comments.
- **Video Description with Latent Topic Guidance:** Topic-guided generation, but for video, not text comments.
- **VREVIEW:** Topic modeling for VR app reviews, focused on topic extraction and summarization.
- **TopicDA:** Topic-conditioned data augmentation for logical form generation, using LLMs.

**Relation to Methodological Clusters:**
- The submission is most closely aligned with the "Topic-Guided Generation" cluster, but distinguishes itself by:
  - Focusing on *interpretable* topic and description generation (not just clustering or summarization).
  - Using LLMs (zero-shot and fine-tuned) rather than probabilistic topic models.
  - Targeting user-contributed comments, a domain with high noise and informality.

**Place in Problem Space and Evaluation:**
- The submission formalizes a *generation* task (topic + description) rather than clustering or summarization.
- Evaluation via semantic similarity to human references is consistent with recent LLM-based generation work, but less common in traditional topic modeling.

**Verification of Author Positioning:**
- The claim that prior work does not generate interpretable topic titles/descriptions from comments is *partially* supported: most prior topic modeling work clusters or summarizes, but some (e.g., TWAG, Explain Me the Painting) generate topic-guided text in other domains.
- The use of LLMs for this specific task is novel within the comment analysis space, but LLM-based topic-guided generation exists in other domains.

# 3. CONTRIBUTION DELTA ANALYSIS

## Claimed Contribution 1: **Formalization of Topic and Description Reasoning Generation from Comments**
- **Most Similar Prior Work:** Topic-driven Reader Comments Summarization, VREVIEW, TWAG (in other domains).
- **Claimed Difference:** Prior work clusters/summarizes; submission formalizes *generation* of interpretable topic titles/descriptions.
- **Independent Assessment:** 
  - MSTM/EXTM and VREVIEW do not generate natural language topic titles/descriptions; they output topic clusters or keywords.
  - TWAG and Explain Me the Painting *do* generate topic-guided text, but not from user comments.
- **Delta:** The formalization of this *generation* task for user comments is a substantive extension within the comment analysis space, but similar generation tasks exist in other domains.

## Claimed Contribution 2: **Automation of Interpretable Topic Title/Description Generation Using LLMs**
- **Most Similar Prior Work:** TopicDA (LLM-based topic-conditioned generation), TWAG (topic-guided generation), Explain Me the Painting.
- **Claimed Difference:** Prior work does not use LLMs for this task on user comments.
- **Independent Assessment:** 
  - LLM-based topic-guided generation is established in other domains (Wikipedia, art, logic forms).
  - Application to user comments is new, but the underlying approach (LLM-based, topic-guided generation) is not unique.
- **Delta:** The main novelty is domain/application-specific, not methodological.

## Claimed Contribution 3: **Fine-tuning LLMs with Human-Annotated Comments Improves Output**
- **Most Similar Prior Work:** TopicDA (fine-tuning for topic-conditioned generation), LLM fine-tuning literature.
- **Claimed Difference:** Demonstrates improvement via fine-tuning on human-annotated comments.
- **Independent Assessment:** 
  - Fine-tuning LLMs for improved generation is a standard practice; the novelty is in the dataset/task, not the method.
- **Delta:** This is a routine adaptation; improvement is expected and not conceptually novel.

## Claimed Contribution 4: **Observation about Context Interference**
- **Most Similar Prior Work:** Not directly addressed in cited works.
- **Claimed Difference:** Providing additional context (captions) can interfere with LLM output.
- **Independent Assessment:** 
  - This is an empirical observation, not a methodological advance.
- **Delta:** Interesting, but not a substantive contribution.

**In summary:**  
The submission's main novelty lies in formalizing and applying LLM-based topic and description generation to user-contributed comments, a task not previously addressed in this domain. However, similar methods and problem formulations exist in other domains (Wikipedia, art, logic forms), and the methodological advances are incremental. The authors' characterizations of prior work are accurate for the cited papers but omit relevant LLM-based topic-guided generation literature, which would provide a more balanced context for their contributions. Fine-tuning LLMs for improved output is a routine adaptation, and empirical improvements may reflect data or implementation details rather than conceptual innovation. Reviewers should consider the degree to which domain adaptation constitutes substantive novelty in this active and rapidly evolving field.
