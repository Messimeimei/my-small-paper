# RESEARCH LANDSCAPE ANALYSIS

## 1. METHODOLOGICAL LANDSCAPE

### Main Methodological Approaches

Across the submission and related papers, the research landscape is characterized by a rich interplay of **data construction/augmentation**, **query reformulation**, **retrieval optimization**, **user simulation**, **multimodal integration**, and **intent modeling**. The dominant methodological trends are:

- **Automated Data Construction & Augmentation**: Leveraging LLMs, graph-based methods, and synthetic techniques to generate large-scale, high-quality conversational datasets (e.g., ICConv, ConvTrans, ConvSDG, Dialog Inpainting, BlendX, MIntRec2.0).
- **Conversational Query Reformulation (CQR)**: Developing models and frameworks for rewriting user queries to be contextually complete and retrieval-effective, often using LLMs and generative PLMs (e.g., AdaCQR, RETPO, ConvGQR, LLM4CS, Enhancing Conversational Search).
- **User Simulation & Feedback Incorporation**: Simulating user interactions and feedback to train and evaluate conversational systems (e.g., ConvSim, LLM-based user simulators).
- **Multimodal and Multi-Intent Modeling**: Integrating text, images, audio, and explicit intent signals for richer conversational understanding (e.g., MMConv, MIntRec2.0, BlendX, Melon).
- **Personalization and Preference Modeling**: Extracting and utilizing user preferences for personalized conversational search (e.g., LAPS).
- **Retrieval System Alignment**: Aligning query reformulation and retrieval models for optimal performance across both sparse and dense retrieval paradigms (e.g., AdaCQR, RETPO).

### Methodological Clusters

**Cluster 1: Data Construction & Augmentation**
- **Papers**: Submission (ICConv), ConvTrans, ConvSDG, Dialog Inpainting, BlendX, MIntRec2.0, LAPS
- **Characteristics**: Focus on scalable, often automated, generation of conversational data using LLMs, graph-based transformations, or inpainting. Emphasis on multi-intent, context, and multimodality.
- **Trends**: Increasing reliance on LLMs for both synthetic data generation and augmentation; movement toward more realistic, diverse, and complex datasets.

**Cluster 2: Query Reformulation & Optimization**
- **Papers**: AdaCQR, RETPO, ConvGQR, Enhancing Conversational Search, LLM4CS, Few-Shot Generative CQR, Open-Domain QA via QR, Leveraging Query Resolution
- **Characteristics**: Methods for rewriting queries to resolve context, ambiguity, and optimize for retrieval. Use of LLMs, generative PLMs, and retriever feedback.
- **Trends**: Shift from human-annotated rewrites to LLM-generated, retriever-aligned, and contextually rich reformulations; integration of query rewriting and expansion.

**Cluster 3: User Simulation & Feedback**
- **Papers**: Exploiting Simulated User Feedback, Analysing Utterances in LLM-Based User Simulation
- **Characteristics**: LLM-based user simulators for generating interaction data and evaluating conversational systems; focus on explicit feedback and clarifying questions.
- **Trends**: LLMs as user proxies; feedback-driven system improvement.

**Cluster 4: Multimodal and Multi-Intent Modeling**
- **Papers**: MMConv, MIntRec2.0, BlendX, Melon
- **Characteristics**: Integration of multiple modalities (text, image, audio) and explicit modeling of multi-intent utterances; annotated datasets for complex scenarios.
- **Trends**: Move toward real-world complexity—multi-party, multi-intent, multimodal, and out-of-scope detection.

**Cluster 5: Personalization and Preference Modeling**
- **Papers**: LAPS
- **Characteristics**: Extraction and use of user preferences for personalized dialogue and recommendation; semi-structured memory for preference storage.

### Methodological Trends & Relationships

- **LLMs as a Unifying Backbone**: LLMs are central to data generation, query rewriting, user simulation, and even intent modeling.
- **From Synthetic to Realistic Data**: Early synthetic datasets are being replaced or augmented by more realistic, context-rich, and multimodal datasets.
- **Alignment with Downstream Tasks**: There is a clear trend toward aligning data generation and query reformulation with retrieval effectiveness, often using retriever feedback (RETPO, AdaCQR).
- **Integration of Multiple Tasks**: Increasingly, methods combine query rewriting, expansion, and intent modeling for holistic conversational search (ConvGQR, LLM4CS).
- **Quality Control and Evaluation**: Automated and human-in-the-loop quality control pipelines are becoming standard in dataset construction (ICConv, LAPS).

## 2. PROBLEM SPACE MAPPING

### Key Problems Addressed

- **Data Scarcity and Quality**: Lack of large-scale, high-quality, context-aware, and multi-intent conversational search datasets (ICConv, ConvTrans, ConvSDG, Dialog Inpainting, MMConv, BlendX, MIntRec2.0).
- **Contextual and Multi-Intent Understanding**: Difficulty in modeling and detecting multiple intents and contextual dependencies in conversations (ICConv, BlendX, MIntRec2.0).
- **Query Reformulation for Retrieval**: Challenges in reformulating under-specified, ambiguous, or context-dependent queries for effective retrieval (AdaCQR, RETPO, ConvGQR, Enhancing Conversational Search, Open-Domain QA via QR).
- **User Simulation and Feedback**: Scarcity of real user interaction data and the need for effective user simulation and feedback incorporation (Exploiting Simulated User Feedback, Analysing Utterances in LLM-Based User Simulation).
- **Multimodal and Multi-Domain Search**: Need for datasets and models that handle multimodal (text, image, audio) and multi-domain conversational search (MMConv, Melon, MIntRec2.0).
- **Personalization**: Lack of scalable, personalized conversational data and methods for extracting and utilizing user preferences (LAPS).
- **Evaluation and Benchmarking**: Need for comprehensive evaluation metrics and benchmarks that reflect real-world conversational search challenges (ICConv, MMConv, BlendX, Leading Conversational Search).

### Approaches to Similar Problems

- **Data Scarcity**: Addressed via LLM-based synthetic data generation (ConvSDG, Dialog Inpainting, LAPS), graph-based augmentation (ConvTrans), and automated pipelines (ICConv).
- **Contextual Understanding**: Tackled by constructing datasets with explicit context and multi-intent annotations (ICConv, BlendX, MIntRec2.0), and by integrating context into query reformulation models (AdaCQR, LLM4CS).
- **Query Reformulation**: Approached through LLM prompting, generative PLMs, retriever-aligned optimization, and hybrid rewriting-expansion frameworks (AdaCQR, RETPO, ConvGQR, Enhancing Conversational Search).
- **User Simulation**: LLM-based simulators generate realistic multi-turn interactions and explicit feedback (Exploiting Simulated User Feedback, Analysing Utterances).
- **Multimodality**: Datasets and models incorporate images, audio, and structured knowledge (MMConv, Melon, MIntRec2.0).
- **Personalization**: LLM-augmented human-in-the-loop data collection and preference extraction (LAPS).

### Patterns in Problem Formulation

- **From Single-Intent to Multi-Intent**: Shift from simple, single-intent queries to complex, multi-intent, and context-dependent utterances.
- **From Text-Only to Multimodal**: Expansion from text-based to multimodal (image, audio, video) conversational search.
- **From Static to Interactive**: Movement from static datasets to interactive, feedback-driven, and user-simulated environments.
- **From Manual to Automated**: Increasing automation in data construction, annotation, and evaluation.

## 3. EVALUATION LANDSCAPE

### Common Datasets

- **Conversational Search Datasets**: ICConv (submission), MS MARCO, CAsT-19/20/21, QReCC, TREC CAsT, MMConv, MIntRec2.0, BlendX, LAPS, Melon, WikiDialog, WebDialog.
- **Intent Detection Datasets**: ATIS, SNIPS, CLINC150, Banking77, Intentonomy.
- **Synthetic/Generated Datasets**: ConvSim transcripts, RF COLLECTION, pseudo-relevance labeled datasets.

### Evaluation Metrics

- **Retrieval Metrics**: nDCG@3/10, Recall, MRR, F1 (for QA), passage ranking metrics.
- **Dialogue/Utterance Quality**: Human evaluation (diversity, coherence, informativeness, adequacy), Dist-n, Ent-n, SELF-BLEU, UniEval.
- **Intent Detection**: Accuracy, F1, OOS detection robustness.
- **Personalization**: Personalization accuracy (matching recommendations to preferences).
- **Custom Metrics**: Usefulness (for question suggestion), fusion metric (for generalization across retrieval systems), custom complexity/diversity metrics (BlendX).

### Patterns in Performance Measurement

- **Retrieval-Centric**: Most works use standard IR metrics (nDCG, Recall, MRR) to measure the effectiveness of query reformulation and retrieval.
- **Human Evaluation**: Increasing reliance on human judgments for dialogue quality, answer adequacy, and personalization.
- **Dataset Analysis**: Statistical and custom metrics to assess dataset complexity, diversity, and coverage (ICConv, BlendX).
- **Alignment with Downstream Tasks**: Some works (RETPO, AdaCQR) directly optimize and evaluate query reformulation for retrieval effectiveness, moving beyond surface-level rewrite quality.

### Comparative Evaluation Approaches

- **Baselines**: Comparison with human rewrites, previous SOTA models, and both sparse and dense retrievers.
- **Ablation Studies**: Evaluating the impact of individual components (e.g., query rewriting vs. expansion, multimodal vs. unimodal).
- **Cross-Dataset Evaluation**: Testing models on multiple datasets to assess generalizability (ConvSDG, AdaCQR).
- **Online A/B Testing**: Real-world user engagement metrics (Leading Conversational Search).

## 4. RESEARCH CLUSTERS

### Cluster 1: **Synthetic Data Generation & Augmentation**
- **Papers**: Submission (ICConv), ConvTrans, ConvSDG, Dialog Inpainting, BlendX, MIntRec2.0, LAPS
- **Characteristics**: Focus on scalable, automated, and diverse data generation for conversational search, often leveraging LLMs and graph-based methods. Emphasis on multi-intent, context, and multimodality.
- **Relationships**: These works provide the foundational data resources that enable advances in modeling, retrieval, and evaluation.

### Cluster 2: **Query Reformulation & Retriever Alignment**
- **Papers**: AdaCQR, RETPO, ConvGQR, Enhancing Conversational Search, LLM4CS, Few-Shot Generative CQR, Open-Domain QA via QR, Leveraging Query Resolution
- **Characteristics**: Methods for context-aware, retriever-optimized query rewriting, often using LLMs, generative PLMs, and feedback-driven optimization.
- **Relationships**: These works build on the data resources from Cluster 1 and focus on bridging the gap between user queries and retrieval system requirements.

### Cluster 3: **User Simulation & Feedback**
- **Papers**: Exploiting Simulated User Feedback, Analysing Utterances in LLM-Based User Simulation
- **Characteristics**: LLM-based user simulators for generating realistic interaction data and explicit feedback, supporting both training and evaluation.
- **Relationships**: These works support both data generation (Cluster 1) and model evaluation (Cluster 2).

### Cluster 4: **Multimodal and Multi-Intent Modeling**
- **Papers**: MMConv, MIntRec2.0, BlendX, Melon
- **Characteristics**: Datasets and models for multimodal (text, image, audio) and multi-intent conversational search, with detailed annotations and complex scenarios.
- **Relationships**: These works extend the scope of conversational search to more realistic, challenging settings.

### Cluster 5: **Personalization and Preference Modeling**
- **Papers**: LAPS
- **Characteristics**: Methods and datasets for extracting and utilizing user preferences for personalized conversational search and recommendation.
- **Relationships**: Complementary to other clusters, focusing on user-centric dialogue modeling.

## 5. TECHNICAL EVOLUTION

### Progression and Evolution of Ideas

- **From Manual to Automated Data Construction**: Early reliance on manual annotation and curation is being replaced by LLM-driven, automated, and scalable data generation (ICConv, ConvTrans, ConvSDG, Dialog Inpainting, LAPS).
- **From Simple to Complex Data**: Datasets have evolved from single-intent, text-only, and single-turn to multi-intent, multimodal, multi-turn, and multi-party (BlendX, MMConv, MIntRec2.0, Melon).
- **From Query Rewriting to Retriever Alignment**: Query reformulation has progressed from mimicking human rewrites to optimizing for retrieval effectiveness using retriever feedback and alignment (RETPO, AdaCQR).
- **From Modular to Integrated Systems**: Movement from modular pipelines (separate query rewriting, retrieval, ranking) to integrated, end-to-end models (Contextualized Query Embeddings, ConvGQR).
- **From Static to Interactive Evaluation**: Evaluation has shifted from static benchmarks to interactive, user-simulated, and feedback-driven settings (ConvSim, Analysing Utterances, Leading Conversational Search).
- **From Text to Multimodality and Personalization**: Expansion from text-based systems to those incorporating images, audio, and explicit user preferences (MMConv, Melon, LAPS).

### Building Blocks and Extensions

- **LLMs as a Foundation**: LLMs are the backbone for data generation, query rewriting, user simulation, and even intent modeling.
- **Retriever Feedback Loops**: RETPO and AdaCQR introduce feedback loops where retriever performance directly informs query reformulation.
- **Hybrid and Fusion Approaches**: Integration of sparse and dense retrieval, query rewriting and expansion, and multimodal fusion (AdaCQR, ConvGQR, MMConv, MIntRec2.0).

### Competing and Complementary Approaches

- **Competing**: Human-annotated vs. LLM-generated data; sparse vs. dense retrieval alignment; unimodal vs. multimodal modeling.
- **Complementary**: Data generation methods (Cluster 1) enable advances in query reformulation (Cluster 2); user simulation (Cluster 3) supports both data and evaluation; multimodal and personalization approaches (Clusters 4 & 5) enrich the conversational search paradigm.

# SUMMARY

The research landscape in conversational search is rapidly evolving, driven by advances in LLMs, automated data generation, and retriever-aligned optimization. The field is moving toward more realistic, complex, and user-centric scenarios, with a strong emphasis on context, multi-intent, multimodality, and personalization. Methodologically, there is a convergence toward integrated, feedback-driven, and scalable approaches, with LLMs serving as a unifying foundation across data, modeling, and evaluation. The submission paper (ICConv) is well-positioned within this landscape, addressing critical gaps in data quality, scale, and context-awareness, and providing a new standard for evaluation and model development in conversational search.
