This paper introduces Arctic-SnowCoder-1.3B, a data-efficient base code model pretrained on 555 billion tokens through a multi-phase process that emphasizes the use of high-quality data, including a phase where a data annotator is employed to select superior code samples for pretraining. While the approach of leveraging a data annotator to extract high-quality data is valuable and contributes to the model’s strong performance on benchmarks such as BigCodeBench, the novelty of this methodology is limited. Similar strategies have already been explored in the literature, notably the use of large language models like GPT-4 as data annotators for curating pretraining datasets. As a result, the uniqueness of the authors’ contribution is somewhat diminished, since the core idea of using automated annotators for data selection is not new. The paper would benefit from more clearly articulating how its approach to data annotation and selection differs from or improves upon these existing methods to better establish its distinct contribution to the field.
