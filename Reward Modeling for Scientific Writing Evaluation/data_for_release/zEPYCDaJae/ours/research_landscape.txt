# RESEARCH LANDSCAPE ANALYSIS

## 1. METHODOLOGICAL LANDSCAPE

### Main Methodological Approaches

Across the collection, the dominant methodological trend is the **automation of data-centric processes** using **Large Language Models (LLMs)** and related AI techniques. The methods can be grouped into the following clusters:

#### **Cluster 1: LLM-Driven Automated Data Processing and Discovery**
- **DataSEA (Submission):** Fully automated framework for dataset processing using LLMs, including autonomous search, retrieval, organization, and script generation.
- **CAAFE:** LLM-based automated feature engineering for tabular data, generating code and explanations.
- **Causal Dataset Discovery:** LLM-based framework for discovering causal links between datasets, with heuristic prompting and fine-tuning.
- **Auctus:** Dataset search engine with advanced query capabilities and data profiling.
- **Dataset Discovery and Exploration (Survey):** Overview of methods for dataset search, navigation, and annotation.

**Commonalities:**  
- Heavy reliance on LLMs for automating traditionally manual data tasks.
- Emphasis on semantic understanding, context-awareness, and code generation.
- Focus on scalability and reducing human effort.

#### **Cluster 2: Automated Dataset Curation, Refinement, and Generation**
- **APIGen:** Automated pipeline for generating and verifying function-calling datasets.
- **SHED:** Automated dataset refinement for instruction fine-tuning using Shapley values and clustering.
- **Biomedical Image Processing Pipeline:** Automated, containerized pipeline for image data analysis.
- **Data-centric AI (Perspectives):** Systematic approaches for data development, maintenance, and evaluation.

**Commonalities:**  
- Automation of dataset creation, curation, and refinement.
- Use of verification, clustering, and quality control to ensure data reliability.
- Focus on improving downstream model performance and efficiency.

#### **Cluster 3: LLMs for Automated Code and Analytic Generation**
- **LLMs for Science:** Empirical study of LLMs for code and data analysis script generation.
- **NL4DV-LLM:** LLM-based translation of natural language queries into analytic specifications for data visualization.
- **ChatGPT for Code Refinement:** Empirical evaluation of LLMs for code refinement based on review comments.

**Commonalities:**  
- LLMs as code and analytic specification generators.
- Evaluation of LLMs’ effectiveness in automating technical tasks.
- Focus on explainability, debuggability, and practical utility.

#### **Cluster 4: Automated Evaluation and Benchmarking**
- **ToolQA:** Automated dataset and benchmark for evaluating LLMs’ tool-use abilities.
- **Pythia:** Suite of LLMs and tools for controlled analysis of training, scaling, and memorization.

**Commonalities:**  
- Creation of new benchmarks and datasets for robust evaluation.
- Emphasis on transparency, reproducibility, and diagnostic capabilities.

#### **Cluster 5: Data Science Pipeline Analysis**
- **Comprehensive Study of Data Science Pipelines:** Systematic analysis of pipeline architectures in theory and practice.

**Commonalities:**  
- Focus on understanding and characterizing the structure and evolution of data science workflows.

### Methodological Trends and Relationships

- **LLMs as Central Enablers:** Most recent methods leverage LLMs for automation, semantic understanding, and code generation, marking a shift from rule-based or manual approaches.
- **Automation and Scalability:** There is a clear trend toward automating every stage of the data pipeline, from discovery and curation to analysis and evaluation.
- **Integration of Verification and Quality Control:** New pipelines (e.g., APIGen, SHED) incorporate multi-stage verification and quality assessment to ensure data reliability.
- **Explainability and Interpretability:** Methods like CAAFE and NL4DV-LLM emphasize generating human-readable explanations alongside code or analytic outputs.
- **Bridging Data and Model-Centric Approaches:** Data-centric AI perspectives and tools like SHED and DataSEA aim to balance improvements in data quality with advances in modeling.

## 2. PROBLEM SPACE MAPPING

### Key Problems Addressed

1. **Automating Data Discovery, Preparation, and Exploration**
   - *DataSEA, Auctus, Dataset Discovery Survey, Causal Dataset Discovery*
   - Problems: Manual, inefficient, and non-scalable data discovery and preparation; lack of expressive search and integration capabilities.

2. **Automated Feature Engineering and Data Augmentation**
   - *CAAFE, Auctus, Data-centric AI*
   - Problems: Incorporating domain knowledge, generating meaningful features, and augmenting data for improved model performance.

3. **Dataset Curation, Refinement, and Quality Assurance**
   - *SHED, APIGen, Data-centric AI, Biomedical Image Processing Pipeline*
   - Problems: Identifying high-quality, non-redundant, and reliable data; reducing computational costs; ensuring data transferability and generalizability.

4. **Automated Code and Analytic Generation**
   - *LLMs for Science, NL4DV-LLM, ChatGPT for Code Refinement*
   - Problems: Generating application code, analytic scripts, and visualizations from natural language or review comments; ensuring correctness and explainability.

5. **Evaluation and Benchmarking of LLM Capabilities**
   - *ToolQA, Pythia*
   - Problems: Faithful evaluation of LLMs’ tool-use, reasoning, and scaling; lack of robust, unbiased benchmarks.

6. **Understanding and Characterizing Data Science Pipelines**
   - *Comprehensive Study of Data Science Pipelines*
   - Problems: Lack of clarity on pipeline architectures and their evolution in theory and practice.

### Approaches to Similar Problems

- **Data Discovery:**  
  - *DataSEA* automates the entire process using LLMs, while *Auctus* focuses on expressive search and integration, and the *Survey* provides a taxonomy of approaches.
  - *Causal Dataset Discovery* extends discovery to causal relationships, using LLMs and heuristic prompting.

- **Dataset Curation:**  
  - *SHED* uses Shapley values and clustering for selection, *APIGen* uses multi-stage verification, and *Biomedical Image Processing* uses containerization and orchestration for scalable, automated processing.

- **Code/Analytic Generation:**  
  - *LLMs for Science* and *NL4DV-LLM* focus on generating code/analytic specifications from natural language, with *NL4DV-LLM* emphasizing explainability.
  - *ChatGPT for Code Refinement* empirically evaluates LLMs for code improvement tasks.

### Patterns in Problem Formulation

- **Scalability and Automation:** Most problems are framed around the need to scale data-centric processes and reduce manual intervention.
- **Quality and Reliability:** There is a strong emphasis on ensuring the quality, reliability, and explainability of both data and model outputs.
- **Bridging Gaps:** Several works aim to bridge gaps between data and model-centric approaches, or between theory and practice in data science workflows.

## 3. EVALUATION LANDSCAPE

### Common Datasets and Evaluation Methods

- **Datasets:**
  - Many works use *custom or newly constructed datasets* (e.g., APIGen, ToolQA, Biomedical images, code review datasets).
  - Some use *public benchmarks* (e.g., MMLU, WizardLM, The Pile, NL4DV Corpus).
  - *Auctus* and *DataSEA* operate over large, diverse, or unspecified collections.

- **Metrics:**
  - **Task-Specific Metrics:**  
    - *Accuracy* (NL4DV-LLM, SHED, Biomedical Image Processing)
    - *ROC AUC* (CAAFE)
    - *Success Rate* (ToolQA)
    - *Exact Match, BLEU* (Code Refinement)
    - *Bias, Memorization Rate* (Pythia)
    - *Fairness* (SHED)
    - *Efficiency, Robustness* (DataSEA)
  - **Benchmark Rankings:**  
    - *Berkeley Function-Calling Leaderboard* (APIGen)
  - **Qualitative/Empirical Analysis:**  
    - *LLMs for Science* and *Comprehensive Study of Pipelines* use empirical or comparative analysis.

### Patterns in Performance Measurement

- **Emphasis on Automation Gains:**  
  - *DataSEA* and similar works measure efficiency in terms of time and effort reduction.
- **Comparative Benchmarks:**  
  - Many works compare against baselines or state-of-the-art (e.g., CAAFE vs. baseline, APIGen vs. GPT-4o).
- **Transferability and Generalizability:**  
  - *SHED* and *APIGen* assess how well curated/generated data transfer across models or tasks.
- **Explainability and Interpretability:**  
  - Some works (CAAFE, NL4DV-LLM) include human-readable explanations as part of their evaluation.

### Comparison of Evaluation Approaches

- **Task-Driven vs. System-Driven:**  
  - Some papers focus on end-task performance (e.g., classification accuracy, code correctness), while others evaluate the system’s ability to automate or scale processes (e.g., time saved, robustness).
- **Benchmark Creation:**  
  - Several works introduce new benchmarks (ToolQA, APIGen, Causal Dataset Discovery) to fill gaps in evaluation.
- **Empirical and Qualitative Analysis:**  
  - Studies like *LLMs for Science* and *Comprehensive Study of Pipelines* provide broader, less quantitative assessments.

- **Cluster 4** provides the evaluation infrastructure necessary to assess advances in Clusters 1–3.
- **Cluster 5** offers a meta-perspective, contextualizing the evolution and integration of methods across the other clusters.

## 5. TECHNICAL EVOLUTION

### Progression and Evolution of Ideas

- **From Manual to Automated:**  
  - Early works and surveys highlight the manual, fragmented nature of data discovery and curation. Recent methods (DataSEA, CAAFE, SHED, APIGen) automate these processes using LLMs and advanced algorithms.
- **From Model-Centric to Data-Centric:**  
  - There is a visible shift from focusing solely on model improvements to systematic data-centric approaches (Data-centric AI, SHED, APIGen).
- **From Simple Search to Semantic Understanding:**  
  - Dataset search has evolved from keyword-based (Auctus) to context-aware, semantic, and causal discovery (DataSEA, CAAFE, Causal Dataset Discovery).
- **From Static to Dynamic and Verified Datasets:**  
  - Dataset generation and curation now include multi-stage verification (APIGen) and dynamic refinement (SHED), ensuring higher quality and reliability.
- **From Black-Box to Explainable Automation:**  
  - Newer methods (CAAFE, NL4DV-LLM) emphasize explainability and interpretability, generating not just outputs but also human-understandable rationales.

### Building Blocks and Extensions

- **LLMs as Universal Engines:**  
  - LLMs serve as the backbone for automation, semantic understanding, and code generation across multiple domains.
- **Verification and Quality Control:**  
  - Multi-stage verification (APIGen) and Shapley-based selection (SHED) extend basic automation with robust quality assurance.
- **Benchmarking and Evaluation:**  
  - New benchmarks (ToolQA, APIGen datasets, Pythia suite) enable more rigorous and targeted evaluation of LLM capabilities.

### Competing and Complementary Approaches

- **Competing:**  
  - Different approaches to dataset curation (e.g., SHED’s Shapley-based selection vs. APIGen’s verification pipeline) offer alternative solutions to similar problems.
  - Dataset search engines (Auctus vs. LLM-based DataSEA) differ in their reliance on metadata vs. semantic LLM understanding.

- **Complementary:**  
  - DataSEA’s end-to-end automation can leverage curated datasets from SHED or APIGen.
  - Evaluation frameworks (ToolQA, Pythia) are essential for assessing the effectiveness of automation and curation methods.
  - Data-centric AI perspectives provide a unifying framework for integrating advances across clusters.

# SUMMARY

The research landscape is rapidly evolving toward **fully automated, LLM-driven data-centric workflows**. The field is characterized by:
- **Automation** of data discovery, curation, and analysis using LLMs.
- **Emphasis on data quality, reliability, and explainability**.
- **Creation of new benchmarks and evaluation frameworks** to rigorously assess progress.
- **Integration of data and model-centric approaches** for holistic AI system development.

**DataSEA** (the submission) exemplifies the latest trend: a comprehensive, LLM-powered system that automates the entire dataset processing pipeline, building on and extending the foundational work in dataset discovery, curation, and automation. The landscape is marked by both **competing methodologies** (e.g., different curation strategies) and **complementary advances** (e.g., new benchmarks enabling better evaluation), all converging toward more efficient, scalable, and reliable data-driven research.
