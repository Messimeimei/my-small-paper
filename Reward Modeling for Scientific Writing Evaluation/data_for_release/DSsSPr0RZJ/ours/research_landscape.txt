# METHODOLOGICAL LANDSCAPE

**Overview:**  
The research landscape is dominated by the development of benchmarks, agent frameworks, and evaluation methodologies for data science automation, code generation, and multimodal agent tasks. The main methodological approaches can be grouped as follows:

### Cluster 1: Comprehensive Benchmarks for Data Science Agents
- **Papers:** DSBench (Submission), DSEval (Related 1), InfiAgent-DABench (3), Spider2-V (4), OSWorld (7), WebArena (6), WorkArena (12), AgentBench (13)
- **Description:**  
  These works focus on constructing large-scale, realistic benchmarks to evaluate the capabilities of LLM-based agents in data science, software, and web environments. They emphasize:
  - Real-world task collection (e.g., ModelOff, Kaggle, StackOverflow, enterprise software)
  - End-to-end workflows (not just code completion)
  - Multimodal and multi-step tasks (code, GUI, web, etc.)
  - Automatic and/or execution-based evaluation protocols
- **Trends:**  
  - Increasing realism and complexity in benchmarks (longer contexts, multimodal, multi-application)
  - Shift from synthetic or simplified tasks to authentic, user-driven problems
  - Emphasis on reproducibility, scalability, and coverage

### Cluster 2: Agent Architectures and Automation Frameworks
- **Papers:** DS-Agent (2), Data Interpreter (5), SheetCopilot (11), WebVoyager (8)
- **Description:**  
  These works propose new agent architectures or frameworks for automating data science or software tasks, often leveraging LLMs with additional reasoning or planning modules:
  - Case-Based Reasoning (CBR) for leveraging expert knowledge (DS-Agent)
  - Hierarchical graph modeling for dynamic workflow management (Data Interpreter)
  - State machine-based planning for software control (SheetCopilot)
  - Multimodal perception and action for web agents (WebVoyager)
- **Trends:**  
  - Integration of LLMs with explicit planning, memory, or reasoning modules
  - Focus on adaptability, robustness, and real-time task management
  - Emphasis on reducing resource requirements (e.g., avoiding finetuning via CBR)

### Cluster 3: Code Generation and NL-to-Code Benchmarks
- **Papers:** BigCodeBench (9), DS-1000 (10), ARCADE (14)
- **Description:**  
  These works develop benchmarks and models for code generation, especially in data science contexts:
  - Realistic, diverse, and complex code tasks (BigCodeBench, DS-1000)
  - Multi-turn, context-rich NL-to-code problems in notebooks (ARCADE)
  - Emphasis on execution-based and multi-criteria evaluation
- **Trends:**  
  - Move toward natural, context-rich, and interdependent code tasks
  - Focus on functional correctness and robustness against memorization

### Cluster 4: Multimodal and Real-World Environment Benchmarks
- **Papers:** Spider2-V (4), OSWorld (7), WebArena (6), WebVoyager (8), WorkArena (12)
- **Description:**  
  These works create environments and benchmarks for agents operating in multimodal, real-world settings (web, desktop, enterprise software):
  - Support for GUI, CLI, and web interactions
  - Multimodal input (text, vision, actions)
  - Execution-based, functional evaluation
- **Trends:**  
  - Increasing focus on multimodal perception and action
  - Emphasis on real computer/web environments over simulations

**Relationships:**  
- Many benchmarks (Cluster 1) serve as testbeds for agent architectures (Cluster 2).
- Code generation benchmarks (Cluster 3) often inform or are integrated into broader agent evaluation frameworks.
- Multimodal benchmarks (Cluster 4) push the boundaries of agent capabilities, requiring advances in both agent design and evaluation.

# EVALUATION LANDSCAPE

**Common Datasets:**
- **Data Science Competitions:** ModelOff, Kaggle (DSBench, DS-Agent, ARCADE)
- **Community Q&A:** StackOverflow (DS-1000)
- **Custom Benchmarks:** DSEval, InfiAgent-DABench, Spider2-V, OSWorld, WebArena, WorkArena, BigCodeBench, SheetCopilot, AgentBench, ARCADE

**Evaluation Metrics:**
- **Task Success Rate / Accuracy:** Most common, often with execution-based or functional correctness criteria (DSBench, InfiAgent-DABench, Spider2-V, OSWorld, WebArena, WorkArena, AgentBench, SheetCopilot)
- **Normalized/Relative Metrics:** RPG (DSBench), comprehensive score (Data Interpreter)
- **Multi-Criteria Evaluation:** Functional correctness + surface-form constraints (DS-1000, BigCodeBench)
- **Cost and Efficiency:** Cost per run (DS-Agent)
- **Human Baselines:** Human performance for comparison (WebArena, OSWorld, BigCodeBench)

**Patterns:**
- Strong trend toward execution-based, functional evaluation over surface-form or static answer matching
- Increasing use of custom, task-specific evaluation scripts and metrics (Spider2-V, OSWorld)
- Normalization and multi-criteria evaluation to handle task diversity and prevent gaming (DSBench, DS-1000)
- Human-in-the-loop or human baseline comparisons to contextualize agent performance

**Comparisons:**
- DSBench introduces RPG for normalized evaluation across heterogeneous tasks, addressing a key gap in prior work
- Many benchmarks (e.g., DS-1000, BigCodeBench) emphasize robustness against memorization and false positives
- Multimodal and real-world environment benchmarks (Spider2-V, OSWorld, WebArena) require execution-based, often custom, evaluation protocols

# TECHNICAL EVOLUTION

**Progression of Ideas:**
- **Early Work:** Focused on code completion and isolated code generation tasks (e.g., HumanEval, MBPP)
- **First Wave of Data Science Benchmarks:** DS-1000, BigCodeBench, ARCADE introduce natural, diverse, and context-rich code/data science tasks with robust evaluation
- **Comprehensive Agent Benchmarks:** DSBench, DSEval, InfiAgent-DABench, Spider2-V, OSWorld, WebArena, WorkArena, AgentBench expand scope to end-to-end workflows, multimodal tasks, and real-world environments
- **Agent Architectures:** DS-Agent, Data Interpreter, SheetCopilot, WebVoyager propose advanced agent frameworks leveraging LLMs with planning, reasoning, and multimodal capabilities
- **Evaluation Methodology Advances:**  
  - Introduction of normalized and multi-criteria metrics (RPG, comprehensive score)
  - Execution-based and functional correctness as standard
  - Human-in-the-loop and human baseline comparisons

**Building Blocks and Extensions:**
- Benchmarks evolve from code-only to multimodal, real-world, and workflow-centric
- Agent architectures build on LLMs, adding memory, planning, and reasoning modules
- Evaluation protocols become more robust, scalable, and reflective of real-world success

**Competing and Complementary Approaches:**
- Some works focus on agent design (DS-Agent, Data Interpreter), others on benchmark construction (DSBench, Spider2-V)
- Complementarity: Agent frameworks are tested on new benchmarks, and benchmarks are refined based on agent performance
- Competition: Different agent architectures (CBR, hierarchical graphs, state machines) vie for best performance on shared benchmarks

# SUMMARY

The research landscape is rapidly evolving from simple code generation and static benchmarks to comprehensive, realistic, and multimodal evaluation of data science agents. The field is characterized by:
- A strong emphasis on realism, diversity, and end-to-end workflows in benchmarks
- The integration of LLMs with advanced planning, reasoning, and memory in agent architectures
- Robust, execution-based, and often normalized evaluation metrics
- A growing focus on multimodal, real-world, and open-ended environments

**DSBench** (the submission paper) stands out by:
- Providing a comprehensive, realistic benchmark for both data analysis and modeling, sourced from real-world competitions
- Introducing the RPG metric for normalized evaluation across heterogeneous tasks
- Revealing the significant gap between current AI agents and expert-level data science performance

This landscape sets the stage for future work on both more capable data science agents and even more challenging, realistic benchmarks and evaluation protocols.
