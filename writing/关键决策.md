|---|---|---|---|---|

| DEC-T02 | 2026-07-17 | 每个样本可调用多个教师，但只接受格式正确且答案等于金标签的轨迹 | 教师也会答错；直接蒸馏会把明显错误写入训练目标，多教师还能提供不同推理路径 | `有效`；正确答案过滤不等于 CoT 质量验证 |
| DEC-T03 | 2026-07-20 | 选择 Qwen3-4B 作为学生基座 | 在本地已测模型中，它在效果、参数规模和推理速度之间最平衡 | `有效但应量化`；SciRM 系列只作为专用模型对照 |
| DEC-T04 | 2026-07-20 | 从两个完整教师构造三套训练数据：DeepSeek、GLM、双教师共同正确 | 分开训练可以测教师差异，共同正确数据可以测一致性筛选的价值 | `有效`；共识数据尚未训练 |

## 4. 方法与科研流程

| ID | 日期 | 决策 | 为什么 | 状态 / 影响 |
|---|---|---|---|---|
| DEC-M01 | 2026-07-24 | Factor mix 权重不得直接解释为专家能力贡献 | A/B 分别加权会产生跨专家交叉项、二次缩放和全局符号对称；等价 LoRA rank 重参数化也可能改变组合 | `有效`；必须用 delta-sum、rank 变换和 leave-one-out 验证机制 |
| DEC-M02 | 2026-07-24 | 搜索目标本身作为待验证变量，不再默认完整 CoT token CE 等于最终评分目标 | positioning_type 已出现 calibration CE 下降而 test macro-F1 下降；长 rationale 还会主导 token-micro loss | `有效`；比较 score-only NLL、example-macro CoT CE 与当前目标 |
| DEC-M03 | 2026-07-24 | 组合方法首先击败最佳缩放单专家和缩放等权组合，再与更复杂方法比较 | 当前 unseen 权重方向接近等权，增益可能来自整体 adapter 强度，而非专家互补 | `有效`；加入 `c*one-hot` 与 `c*equal` 一维搜索 baseline |
| DEC-W01 | 2026-07-24 | 采用 G0--G6 阶段门和 WIP=2 的科研流程 | 当前同时扩任务、专家、教师和路由会放大实验组合，却不能更快建立因果链 | `有效`；G3 前停止新增专家、教师和复杂路由 |
| DEC-W02 | 2026-07-24 | 主任务固定为 actionability 因果锚点、两个 revision transfer target；novelty 探索，meta_reviewer 压力测试 | 让每个任务只承担一个清晰的证据角色，避免用最复杂任务绑架主论文 | `有效`；详见科研流程 |
| DEC-W03 | 2026-07-24 | 结果进入论文前必须绑定 EXP-ID、预先冻结指标与停止条件 | 防止看完结果后更换主指标、比较对象或贡献叙事 | `有效`；每两周做 continue / pivot / stop Gate Review |
