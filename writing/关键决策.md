## 3. 蒸馏与训练

| ID | 日期 | 决策 | 为什么 | 状态 / 影响 |
|---|---|---|---|---|
| DEC-T01 | 2026-07-17 | 先用 `rw_gen/coherence` 打通清洗、蒸馏、训练、评测的完整流水线 | 先限制为一个明确 aspect，便于定位 prompt、数据和训练代码问题 | `有效`；其他专家尚未打通 |
| DEC-T02 | 2026-07-17 | 每个样本可调用多个教师，但只接受格式正确且答案等于金标签的轨迹 | 教师也会答错；直接蒸馏会把明显错误写入训练目标，多教师还能提供不同推理路径 | `有效`；正确答案过滤不等于 CoT 质量验证 |
| DEC-T03 | 2026-07-20 | 选择 Qwen3-4B 作为学生基座 | 在本地已测模型中，它在效果、参数规模和推理速度之间最平衡 | `有效但应量化`；SciRM 系列只作为专用模型对照 |
| DEC-T04 | 2026-07-20 | 从两个完整教师构造三套训练数据：DeepSeek、GLM、双教师共同正确 | 分开训练可以测教师差异，共同正确数据可以测一致性筛选的价值 | `有效`；共识数据尚未训练 |
| DEC-T05 | 2026-07-20 | 使用 TRL conversational prompt-completion 格式，只对 completion 计算 loss | prompt 和 few-shot 是条件，不应作为模型需要复现的监督目标 | `有效` |
| DEC-T06 | 2026-07-20 | Qwen3 训练和推理关闭原生 thinking，使用数据中的显式 `<reasoning>` | 避免原生隐藏思考协议与蒸馏出来的可见推理格式叠加 | `有效` |
| DEC-T07 | 2026-07-20 | 数据按标签分层并固定 ID 划分，配置写入 YAML，训练 seed 固定为 42 | 让不同教师和后续专家尽量可复现，避免每次随机划分改变结果 | `有效`；公平教师比较还需统一样本 ID |
| DEC-T08 | 2026-07-20 | LoRA 首轮配置固定为 `r=16, alpha=32, dropout=0.05, all-linear`，训练 3 epochs | 先建立可工作的单专家基线，再决定是否搜索超参 | `暂定`；尚未做超参敏感性实验 |
| DEC-T09 | 2026-07-20 | 以最低 validation loss 保存最佳 adapter，并按 epoch 保留 checkpoint | 既便于恢复训练，也能观察训练轮数与迁移 / 干扰之间的关系 | `有效`；DeepSeek 当前最佳为 step 410 |
