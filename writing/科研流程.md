# 科研流程

更新时间：2026-07-24。

当前阶段：`G0 研究契约冻结`。当前 WIP 上限为 2 个实验族；在 G3 完成前，不新增教师、不扩充专家数量、不开发复杂路由。

本文件是项目的唯一当前路线图。研究问题及判决见 [研究问题.md](研究问题.md)，已完成实验及数字见 [实验记录.md](实验记录.md)，路线变更见 [关键决策.md](关键决策.md)，已被证据推翻的认识见 [踩坑记录.md](踩坑记录.md)。

本流程默认训练集、校准集和测试集相互独立，不再围绕数据重合展开研究审查。

## 1. 一句话研究契约

> 在相同目标监督和计算预算下，检验面向科学写作评价的 criterion-specific CoT adapters 是否形成可复用能力，以及稳健的适配器组合能否在新评价准则上超过专家选择、简单融合和直接目标适配。

研究对象：rubric-conditioned scientific writing evaluation。

学生模型：Qwen3-4B；冻结主干，以 LoRA 承载准则特定适配。

当前方法：目标任务级 few-shot calibration + LoRA composition。当前不是 zero-shot，也不是样本级动态路由。

默认论文路线：先做严格实证研究；只有机制实验产生并解决新的稳定缺陷时，才升级为方法论文。

## 2. 证据生产顺序

```text
来源论文与任务定义
        |
        v
G0 研究契约：主张、任务、指标、预算、阈值冻结
        |
        v
G1 CoT 因果价值 ----失败----> 删除 reasoning 增强主张
        |
        v
G2 专家跨任务结构 ----失败----> 仅称 task adapters
        |
        v
G3 组合基线决战 ----失败----> 转向 expert selection / 严格负结果
        |
        v
G4 数学机制稳健 ----失败----> delta-space 或显式因子对齐
        |
        v
G5 多 seed、k 曲线、保持能力
        |
        v
G6 成本核算、论文主张冻结
```

任何论文主张不得高于已通过的最低 Gate。一次相对 base 的提升不能同时证明 CoT、迁移、组合和抗干扰。

## 4. 任务角色与指标契约

### 4.1 任务角色

| 角色 | 任务 | 用途 | 是否决定主论文 |
|---|---|---|---|
| CoT 因果锚点 | `actionability` | 当前单专家收益明显，适合最先做训练消融 | 是 |
| Seen control | `positioning_check`、`positioning_type` | 检查本任务专家、选择和强度缩放 | 否，作为机制控制 |
| 首要 transfer target | `revision_relatedness` | 当前 factor mix 相对 base 的 macro-F1 增益最稳定 | 是 |
| 次要 transfer target | `revision_correctness` | 检查主结论能否跨相关目标复现 | 是 |
| 探索性 target | `novelty` | 检查不同任务形态上的可迁移性 | 否，除非结果跨 seed 稳定 |
| 外部压力测试 | `meta_reviewer` | 检查长上下文和复杂输出条件下的失效边界 | 否 |

### 4.2 指标

- 二分类或名义多分类的主指标：macro-F1；次指标：Accuracy、逐类 recall、格式有效率。
- 1--5 有序评分任务增加 MAE、within-one Accuracy 和 quadratic weighted kappa；不能只用 Accuracy / macro-F1。
- 测试差异使用样本级 paired bootstrap；训练和 calibration 稳健性分别由独立 seed 衡量。
- 相同 seed 的多次 greedy rollout 只诊断推理非确定性，不作为独立重复实验。
- G0 暂定最小有意义效应为：相对最强公平 baseline 的 macro-F1 至少 `+1.0 pp`，且主要 calibration seeds 方向一致。正式开跑前必须冻结或修改该阈值。

### 4.3 监督与计算预算

所有组合与适配 baseline 必须明确并尽量对齐：

- 每类 K-shot 还是总计 K-shot；
- calibration 中使用 label、answer 还是完整 CoT；
- 权重搜索的候选次数；
- target-only baseline 的更新步数和可训练参数；
- 教师调用、GPU-hours、峰值显存和推理延迟。

## 5. 阶段门

| Gate | 必须回答的问题 | 必须产物 | 通过标准 | 未通过时 |
|---|---|---|---|---|
| G0 研究契约 | 论文究竟比较什么 | 一句话主张、RQ、主任务、主指标、预算、阈值、实验登记 | 每个计划实验能映射到一个 RQ 和一个决策 | 删除无法映射的实验 |
| G1 CoT 因果 | 收益是否来自 CoT | score-only / answer-only / full-CoT / 无关 rationale，3 train seeds | full-CoT 相对 answer-only 达到冻结阈值且方向稳定 | 删除 reasoning 增强主张 |
| G2 能力复用 | 专家是否超越任务表面模式 | 完整 expert x target 矩阵、能力依赖表、negative control | 至少一个预注册 off-diagonal 正迁移成立 | 统一称 task adapters |
| G3 组合价值 | 组合是否优于选择和简单融合 | best-single、scaled-single、scaled-equal、equal delta、searched factor、searched delta、target-only、ICL | 至少两个 transfer targets 上跨 calibration seeds 超过简单基线 | 停止扩专家和路由；转向选择或负结果 |
| G4 机制稳健 | factor mix 是否依赖 LoRA 坐标，搜索目标是否对齐最终指标 | rank permutation / rotation、A/B rescaling、cross-term、leave-one-out、score-only 与 CoT CE 目标对照 | 等价参数化下结论稳定，且优化目标与最终评分一致；或替代方法显著消除问题 | 不解释 factor 权重；改用 delta-space / alignment 或 score-aligned objective |
| G5 鲁棒与保持 | 结论能否复现、是否减少干扰 | 3 train seeds、10 calibration seeds、k=1/3/5、固定策略 retention matrix | 主效应超过阈值且多数 seed 同向；最差任务回退受控 | 收窄 claim，不新增任务 |
| G6 效率与写作 | 方法是否值得、论文是否闭环 | 成本表、主表、消融表、claim-evidence map、限制 | 每条摘要主张都对应冻结实验 | 不投稿方法型论文 |

### 5.1 Go / No-Go 阈值

| 主张 | GO | NO-GO |
|---|---|---|
| 组合价值 | 相对 `best-scaled-single / scaled-equal / delta-sum` 中的最强者，在两个 revision 主目标上均达到至少 `+1.0 pp` macro-F1，paired-bootstrap 95% CI 下界大于 0，且至少 8/10 calibration seeds 为正 | 只超过 base，或与缩放/简单组合持平 |
| CoT 因果价值 | full-CoT 相对 answer-only 在 3 个训练 seed 的平均增益至少 `+1.0 pp` macro-F1，至少 2/3 seeds 为正，迁移平均的 paired CI 下界大于 0 | full-CoT 不稳定优于 answer-only |
| 能力复用 | 预注册的非对角迁移跨 seed 复现，且 leave-one-out 影响符合能力依赖表 | 迁移矩阵近似纯对角，或无关专家同样有效 |
| Factor 机制 | 等价 rank 变换后主结论不变，且当前 factor 显著优于参数化不变的 delta 对照 | 性能随任意 rank 基变化，或 delta-sum 已达到同等结果 |
| Meta reviewer 入主表 | 格式有效率至少 `0.95`，且主指标超过常数与单专家基线 | 格式有效率未达阈值；继续只作压力测试 |

## 6. 最小公平基线矩阵

每个 transfer target 使用相同 calibration 样本、测试协议和可比预算：

| 优先级 | 方法 | 回答的问题 |
|---|---|---|
| 必须 | Base | 是否存在任何适配增益 |
| 必须 | 每个 single expert | 哪个专家本身能迁移 |
| 必须 | Calibration-best-single / one-hot | 搜索是否只是专家选择 |
| 必须 | Best scaled single：对每个 `c * one-hot` 做一维搜索 | 自由搜索是否只是缩放一个专家 |
| 必须 | Equal factor mix | 搜索权重是否必要 |
| 必须 | Scaled equal factor：`c * (1/n)` 一维搜索 | unseen 增益是否只是等权 soup 的整体强度调节 |
| 必须 | Equal delta-sum | 简单参数增量组合是否足够 |
| 必须 | Searched factor mix | 当前方法 |
| 必须 | Searched delta-sum | factor cross-term 是否必要 |
| 机制对照 | score-span NLL / example-macro CoT CE / token-micro CoT CE | calibration 目标是否与最终评分指标一致 |
| 必须 | 同 K-shot target-only LoRA 或轻量分类头 | 已有专家复用是否优于直接目标适配 |
| 必须 | 同 K-shot ICL | 参数组合是否优于最简单目标监督利用方式 |
| 论文主表 | Joint multi-task LoRA | 模块化专家相对来源论文统一训练思路是否值得 |

只有 `searched composition > best scaled single + scaled equal + delta-sum + 同预算 target adaptation`，才能把收益归因给组合机制。

## 9. 中期路线选择

- 若 searched factor mix 通过 G3、但未通过坐标稳定性：只做 coordinate-invariant delta composition 或显式 rank alignment 其中一条。
- 若 factor mix 未通过 G3：转向 calibration-based expert selection，或把论文定位为 adapter composition 的严格边界研究。
- 若 CoT 未通过 G1：保留模块化 adapter 研究，但删除 CoT reasoning 叙事。
- 若 G2 只有对角线收益：保留 criterion/task experts，不再构造“推理原子”故事。
- 不同时开发因子对齐和无标签 router；两者分别需要完整方法论文级证据。

## 10. 导师式运行规则

1. 每周一只允许提出最多两个待决策问题，不用“多跑几个看看”替代假设。
2. 任何时刻最多两个实验族处于 `running`；新实验必须等待一个已有实验给出判决。
3. 每周五做一次反方审稿：先写最强替代解释，再判断现有对照能否排除它。
4. 每两周必须输出一次 `continue / pivot / stop`，并同步 [关键决策.md](关键决策.md)。
5. 每个结果先进入实验记录并绑定 EXP-ID，之后才能进入论文叙述。
6. 每张论文表必须标明它支持哪个 Claim；没有 Claim 对应关系的表不进入主文。
