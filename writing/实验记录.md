# MoDE 实验记录

更新时间：2026-07-24 01:28 UTC。

本记录以该时刻的 `MoDE/eval_outputs`、对应 MoDE run artifact、训练 manifest 和实际数据文件为准。当前目录有 22 个实验目录、22 份 `metrics.json`、22 份 `predictions.jsonl`。全部结果均已对照预测明细重算，保存的 Accuracy、macro-F1、格式有效率和逐类指标可精确复现。相对 07-23 快照，新增 `meta_reviewer` 的 Qwen3-4B base 与 n=4 factor-mix 评测，且 actionability / revision correctness 的 MoDE 预测明细已补齐。

按 2026-07-24 的研究审查边界，训练集、校准集和测试集的独立性视为已确认前提；本记录不再把数据重合问题作为方法路线的判断依据。

## 1. 先给结论

当前结果能支持的最窄结论是：

> 在一个训练 seed、一个 calibration seed 和目标任务监督 calibration 下，三个任务型 LoRA 的 target-specific factor mix 相对 Qwen3-4B base 在 actionability、positioning、novelty 和两个 revision aspect 上取得描述性提升；其中 revision relatedness 仍是目前最可靠的正向结果。把 coherence 专家并入后的 n=4 mix 在 meta_reviewer 上几乎无效：Accuracy/F1 远低于多数类基线，格式有效率只有约 0.09–0.14。现有证据不能证明多专家协同、真正的 unseen/zero-shot 泛化、可组合推理原子或复杂科学推理能力提升。

最可能的拒稿意见可以压缩成一句话：

> 该工作使用目标任务标签为每个测试任务单独搜索 task-LoRA 的全局系数；seen 任务上的解退化为本任务单专家且融合不优于单专家，新目标任务又缺少最佳缩放单专家、缩放等权组合、delta-sum 和同等监督预算 target adaptation 等对照，更难的 meta_reviewer 尚未形成可用性能。

因此，当前实验不是“结果还不够多”，而是核心因果链尚未成立：

1. 现有专家按任务 aspect 构造，不是按可观测、可复用的推理原子构造。
2. 现有 seen 结果支持专家选择或强度缩放，不支持专家协作。
3. novelty、revision 和 meta_reviewer 都是带目标任务监督的适配，不是 zero-shot 迁移。
4. 完整 CoT completion CE 与最终 score 指标是否一致尚未验证，positioning_type 已出现 CE 更低但 test F1 更差。
5. Factor mix 依赖 LoRA 因子坐标并产生跨专家交叉项，现有权重不能解释为能力贡献。
6. meta_reviewer 首先是输出格式/长上下文协议失败，其次才谈得上路由或迁移。

## 2. 论文来源与本项目的实际关系

方法来源是 [MoDE-CoTD](../paper/垂域微调与推理能力/论文/MoDE-CoTD%20Chain-of-Thought%20Distillation%20for%20Complex%20Reasoning.pdf)，任务和数据来源是 [Reward Modeling for Scientific Writing Evaluation](../paper/面向科学文献跨学科推理的模型微调方法研究/论文/Reward%20Modeling%20for%20Scientific%20Writing%20Evaluation.pdf)。本项目不是两篇论文设置的直接复现，而是把前者的 factor mix 迁移到后者的科学写作评价任务。

### 2.1 MoDE-CoTD 与当前设置的差异

| 项目 | MoDE-CoTD 论文 | 当前完成的 n=3 实验 |
|---|---|---|
| 学生模型 | Flan-T5 250M/780M/3B | Qwen3-4B |
| 专家数 | 10 个任务专家 | 3 个任务专家 |
| 任务 | 算术、常识、时序、符号、逻辑 | Related Work + RevUtil |
| few-shot 含义 | 目标任务总计 N 条 | 每类 5 条；二分类共 10 条，五分类共 25 条 |
| 优化器 | 正文写 Shiwa | Nevergrad NGOpt，budget=40 |
| 权重范围 | `[-1.5, 1.5]` | `[-3, 3]` |
| L1 | `0.05 * sum(abs(w))` | 相同，按每个 run 的 resolved config 确认 |
| 目标 | 少量目标样本上的任务输出 CE | 完整教师 CoT completion 的 token CE |
| LoRA | `r=16`，其余披露不完整 | `r=16, alpha=32, dropout=0.05, all-linear` |

论文自己的专家数消融显示 3 个专家比 10 个专家更不稳定。因此当前 n=3 结果不能外推成对 MoDE-CoTD 核心结论的复现。

### 2.2 任务构念与研究范围

数据源论文适合支持“rubric-conditioned scientific writing evaluation”，但多数任务不是事实型复杂推理：

- `positioning_type` 混合两种互补 rubric：“每段都定位本文”和“仅最后一段集中定位本文”。同一文本在两种 rubric 下标签相反可能是任务定义，不一定是错标。
- `positioning_check` 又混合“单段是否提到本文贡献”和“final paragraph 是否回应 context”两种输入形态，不能直接等同为一个稳定的“定位推理原子”。
- RevUtil 的 actionability、grounding、verifiability、helpfulness 互相相关；输入没有完整论文，主要测评论措辞、具体程度和论证形式，不足以验证评论中的科学主张是否真实。
- Novelty 的 gold 是“人工摘要 verdict 与 GPT-5.1 从 LLM 摘要推断的 verdict 是否一致”，测的是结论对齐，不是论文是否真正新颖。
- Revision 主要测给定修改指令下的 relatedness 和 correctness；数据源论文也明确指出它比 novelty 更偏 instruction-following。
- meta_reviewer 是 10 类审稿意见质量判断，输入含较长论文上下文；当前 base/MoDE 都难以稳定产出合法 `<score>`。

所以目前“复杂推理”“证据验证”“跨学科”都是超出数据构念的表述。更准确的研究对象是“英文 NLP/ML 科学写作中的 rubric-conditioned 判断”；meta_reviewer 本应更接近复杂审稿推理，但现有协议下尚未形成可解释的评测信号。

## 3. 实验身份与协议

### 3.1 专家集合

主表 n=3 实验使用三个 DeepSeek-v4-pro 正确答案过滤 CoT 蒸馏得到的 Qwen3-4B LoRA；meta_reviewer 额外并入 coherence 专家，构成 n=4：

| 专家 | 蒸馏数据 | gradient train / validation | 训练 seed | 用途 |
|---|---|---:|---:|---|
| `actionability_deepseek` | actionability 1,788 条 | 1,609 / 179 | 42 | n=3 与 n=4 |
| `positioning_check_deepseek` | positioning_check 2,666 条 | 2,399 / 267 | 42 | n=3 与 n=4 |
| `positioning_type_deepseek` | positioning_type 944 条 | 850 / 94 | 42 | n=3 与 n=4 |
| `coherence_deepseek` | coherence DeepSeek 蒸馏 | 3,266 / 363 | 42 | 仅 meta_reviewer n=4 |

四个 run 都训练 3 epochs，冻结 Qwen3-4B 主干，只训练约 33.0M LoRA 参数。教师最终分数必须等于 gold，但没有验证 reasoning 是否忠实、充分或引用了真实输入证据。

有一个必须修复的 provenance 错位：当前 `positioning_type` 专家是 DeepSeek 蒸馏专家，但 MoDE 的 positioning_type calibration 文件来自 GLM-5.2 数据及其 validation split。该结果不是“DeepSeek 专家原 validation 上调权”的干净实验。

meta_reviewer 的 calibration 另有教师错位：`api_validation_k5_per_class_cal45_seed42.json` 的教师是 MiniMax-M3，不是专家蒸馏所用的 DeepSeek-v4-pro；且标签 2/5 可用样本不足，k=5 实际只有 45 条（2 只有 2 条、5 只有 3 条）。test 中标签 2 和 5 的 support 均为 0，因为可用样本几乎都被抽进 calibration。

### 3.2 Factor mix

每个 target 都独立运行一次权重搜索并保存不同 adapter：

```text
A_hat = sum_i(w_i * A_i)
B_hat = sum_i(w_i * B_i)
delta_hat = (alpha / r) * B_hat @ A_hat
objective = teacher_CoT_completion_CE + 0.05 * sum(abs(w))
```

这不是 `sum_i(w_i * delta_i)`。展开后包含 `w_i * w_j * B_i @ A_j` 交叉项，并有全局符号对称性 `f(w)=f(-w)`。因此：

- 单专家权重 `0.789` 的自身项尺度约为 `0.789^2=0.622`，不是线性 0.789。
- novelty 的三个负权重与三个权重全部取正得到完全相同的 adapter，不能解释为“反向使用专家”。
- 独立 LoRA 的 rank 轴没有天然对齐；对同一个 LoRA delta 做等价的 rank permutation、rotation 或 A/B rescaling，会保持单专家 `B @ A` 不变，却改变跨专家交叉项。当前 factor mix 对 LoRA 参数化坐标敏感，专家权重不具稳定的能力解释。

### 3.3 评测协议

除 meta_reviewer 外，现有 metrics 使用：`seed=42`、5 次 rollout、greedy decoding、`temperature=0`、`top_p=1`、`max_model_len=8192`、`max_tokens=512`、`batch_size=64`、`thinking=False`。

meta_reviewer 因长 prompt 将 `max_model_len` 提到 `32768`，其余解码超参相同；calibration 序列均值约 15.3k token（范围约 10k–25k），优化时 `objective_max_length=32768`。

五次 rollout 使用相同模型、相同 seed、相同测试集和 greedy decoding，不是五个独立实验。不同任务仍有样本发生跨 rollout 分数变化，这反映 vLLM/GPU 推理非确定性；表中的 `+/- std` 不能当作训练 seed 方差、抽样置信区间或显著性证据。

## 6. 本轮研究审查边界

按当前确认前提，训练集、校准集和测试集相互独立。本轮不再围绕数据重合或数据版本讨论研究路线；后续判断只基于任务构念、方法归因、公平 baseline、统计稳健性、机制解释和成本。

## 7. 对研究思路的拷打

### 7.1 “推理原子”目前只是命名，没有实验对象

当前 actionability、positioning_check、positioning_type 是 task/aspect 标签，分别改变了输入格式、rubric、标签空间、文本来源和训练数据。没有中间状态标注，也没有证明同一个原子跨两个表面不同任务复用。把三个 task-LoRA 叫作“证据验证、定位归因、标准遵循专家”，不会自动让它们成为推理原子。

要让这个科学假设可证伪，至少需要：同一原子在两个不同任务上的操作化定义、对应的可观测错误类型、共享原子但不同表面任务的 transfer test，以及不共享原子的 negative control。

### 7.2 当前 MoDE 证明的是 target adaptation，不是 unseen generalization

novelty、两个 revision aspect 和 meta_reviewer 都用目标任务 gold label 加教师 CoT 做 calibration，再运行 40 次权重候选评估。与完全不使用目标监督的 base 比较，监督预算不公平。公平 baseline 至少包括：

- 同样 K 条 target 样本的 in-context learning；
- calibration 上选择 best single expert；
- equal-weight mix 和随机权重；
- 同样 K 条数据的 target-only LoRA 或轻量分类头；
- joint multi-task LoRA；
- delta-sum/LoRAHub 类线性 adapter composition。

没有这些对照，“+7.27 pp novelty”只能归因于“拿了目标标签后做适配”，不能归因于推理迁移。meta_reviewer 甚至连“适配成功”都谈不上：45 条目标监督之后，test Accuracy 仍只有 0.0358。

### 7.3 现有 seen 结果反而不支持专家协作

三个 seen target 的最优解都接近本任务 one-hot；mix 相对单专家分别约为 +0.06、-0.40、-1.47 pp Accuracy。actionability 的全部主要收益来自单专家，另外两个 mix 更差。继续把这一组结果写成“多专家协同”会被表格本身反驳。

### 7.4 CoT 还没有证明是收益来源，更没有证明 reasoning 变强

教师答对过滤只验证最后一个 `<score>`，不验证中间 reasoning。源数据论文的 GRPO reward 也只依赖格式、长度和最终分数，没有逐步真值。因此当前可见 reasoning 很可能是 post-hoc explanation。

缺少的关键消融是：gold-label only、answer-only、完整 CoT、等 token 无关 rationale，以及 MoDE 中 score-only CE 对完整 CoT CE。当前优化目标的大量 token 来自教师文风，`<score>` 只占极少数 token；优化器可能在拟合 explanation style，而不是判别边界。

### 7.5 搜索目标与最终评分目标发生错位

当前优化器最小化完整教师 CoT completion 的 token-micro CE，但最终只评价 `<score>`。positioning_type 已给出直接反例：本任务 one-hot 专家的 calibration objective 为 `0.681712`，搜索解降到 `0.644368`；但 test macro-F1 从单专家 `1.0000` 降到 mix `0.9836`。搜索目标变好，并没有带来最终任务指标变好。

完整 CoT CE 还会让较长 rationale 在目标中占据更多权重。必须比较 token-micro full-CoT CE、每样本等权 full-CoT CE、score-span NLL 和直接标签概率目标；在该对照完成前，不能把“更低 calibration CE”当作更好的组合证据。

### 7.6 Factor mix 的数学语义不稳

逐因子混合会产生独立 LoRA rank 轴之间的交叉项，且不具 LoRA 等价重参数化不变性。权重热图不能作为专家能力解释。若方法贡献仍依赖 factor mix，必须补：

- 直接 `sum_i(w_i * delta_i)` 的 delta-sum 对照；
- 对每个 LoRA 做随机 rank permutation/rotation 后的稳定性测试；
- shared initialization 或显式 rank alignment；
- leave-one-expert-out 和 cross-term ablation。

如果一个数学上等价的 LoRA 重参数化会改变组合模型性能，那么所谓“专家互补”可能只是坐标碰巧对齐。

### 7.7 指标与统计设计不够

- 目前只有一个训练 seed、一个 calibration seed、一个专家集合和每类 k=5。
- 5 次相同 seed greedy rollout 是伪重复，不能代替训练/数据重复。
- RevUtil 的 1-5 分是有序标签，Accuracy 和 macro-F1 把错 1 分与错 4 分视为同样严重。应增加 MAE、within-one Accuracy、quadratic weighted kappa 和 ordinal confusion matrix。
- 所有任务应报告 majority baseline、逐类 recall、配对 bootstrap CI；格式错误还应拆成 parse failure、截断、重复标签和分值越界。
- 多任务、多模型、多 seed 同时比较时要预注册主指标和主任务，避免结果挑选。

### 7.8 研究贡献目前容易落在“应用现有方法到新数据集”

MoDE 是现成方法，SciRM tasks 是现成任务框架。仅把 MoDE 从通用推理搬到科学写作评价，方法新颖性很弱。若想做方法论文，需要至少解决一个 MoDE 本身没解决的问题，例如 LoRA 因子对齐、无需 target label 的路由、可验证的能力分解或更低成本的组合搜索。否则应诚实定位成严格实证研究，而不是新路由方法。

“减少能力干扰”也尚未被测试。冻结 base 只说明主干没改，不说明启用 mix 后不会伤害其他任务；需要同一个固定模型上的 retention matrix 和 joint fine-tune baseline。

## 8. 当前 claim 的判决

| 候选 claim | 当前判决 | 原因 |
|---|---|---|
| 单任务 CoT-LoRA 能提升部分任务 | 初步支持 | actionability 提升明显；positioning 两项也高于 base，但仍需多训练 seed |
| 三专家融合优于单专家 | 不支持 | 三个 seen target 持平或更差，权重近 one-hot |
| 能力发生跨任务协作 | 不支持 | 缺专家 x 任务迁移矩阵、best-one-hot 和 leave-one-out |
| unseen/zero-shot 泛化 | 错误表述 | 每个 unseen target 都使用监督 calibration |
| 复杂科学推理提升 | 不支持 | 多数任务是 rubric/形式判断，CoT 无忠实性监督；meta_reviewer 近乎格式崩溃 |
| 推理原子可组合 | 未被操作化 | 当前专家按 task aspect 划分，没有原子级标签或 negative control |
| 减少灾难性遗忘/能力干扰 | 未测试 | 没有固定组合模型的全任务 retention 对照 |
| 成本更低 | 未测试 | 未计教师生成、多个 LoRA 训练及每 target 40 次搜索成本 |
| 跨学科泛化 | 不支持 | 当前材料主要来自英文 NLP/ML，缺跨学科分层结果 |
| n=4 专家可解决更难的 meta-reviewer | 不支持 | Acc/F1 远低于多数类，格式有效率 <0.15 |

## 9. 下一轮实验的止损顺序

### P0：补齐能决定论文是否成立的 baseline

对每个 target 固定同一测试集，至少比较 `Base / 每个单专家 / calibration-best-one-hot / best-scaled-single / scaled-equal / equal-delta / searched factor / searched delta / joint multi-task LoRA / 同预算 K-shot target adaptation / ICL`。同时比较 score-only、example-macro CoT 和当前 token-micro CoT 搜索目标。先形成完整的 expert x task transfer matrix，再谈路由。

### P0：meta_reviewer 先修协议，再谈专家数

在格式有效率仍低于 `0.95` 之前，不要继续增加专家或扩大 meta_reviewer 的权重搜索。优先做：强制 score 输出、缩短/分层 prompt，以及 majority / ICL / 单专家对照。当前 0.03 级 Accuracy 不能进入主结果表的正向叙述。

### P1：把随机性放在正确层级

- 至少 3 个训练 seed；
- 至少 10 个 calibration 抽样 seed；
- 分别报告 k=1/3/5，且明确是 total K 还是 per-class K；
- 测试使用样本级 paired bootstrap；greedy 重复仅用于诊断非确定性，不作为独立样本。

### P1：证明收益来自 reasoning，而非标签和格式

做 label-only、answer-only、full-CoT、无关等长 rationale，以及 score-only CE 对 full-completion CE。人工分层审计 reasoning 的证据支撑、忠实性、模板化和 hallucination；增加同一文本切换 rubric 的反事实测试。

### P2：决定是否继续 factor mix

只有当 searched mix 在至少两个预注册的新目标任务上，跨 calibration seeds 稳定超过 best-scaled-single、scaled-equal、delta-sum 和同预算 target baseline，才保留“可组合专家”主张。否则将结果定位为专家选择或任务级适配，并停止增加更复杂路由。

同样，若 full-CoT 在同预算下不能稳定超过 answer-only，就停止使用“推理蒸馏提升 reasoning”这一表述。
