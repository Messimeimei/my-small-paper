# MoDE 实验记录

更新时间：2026-07-23。本文只整理 `MoDE/eval_outputs` 中可复核的结果；每个完成实验均包含 `metrics.json` 和 `predictions.jsonl`。

## 1. 实验目的

评估两个 DeepSeek-v4-pro 蒸馏的 Qwen3-4B LoRA 专家经 MoDE factor mix 后，在目标任务和未参与权重搜索的任务上相对 Qwen3-4B 基座的效果。

融合专家：

- `positioning_check_deepseek`
- `positioning_type_deepseek`

## 2. MoDE 设置

| 项目 | 设置 |
|---|---|
| 基座 | Qwen3-4B |
| 融合方式 | `A_hat = sum(w_i A_i)`，`B_hat = sum(w_i B_i)` |
| 权重搜索任务 | `positioning_check` |
| Calibration | 每类 5 条，共 10 条；使用教师 CoT completion |
| 优化目标 | completion-only token CE + `0.05 * sum(abs(w))` |
| 优化器 | Nevergrad NGOpt，budget 40，权重范围 `[-3, 3]` |
| 最优权重 | positioning_check: `0.995294`；positioning_type: `-0.001690` |
| 最优目标值 | `0.650359` = CE `0.600510` + L1 `0.049849` |
| 随机种子 | 42 |

搜索结果表明，最终融合几乎等同于 positioning_check 专家，positioning_type 专家的权重接近 0。完整搜索产物见 [summary.json](../MoDE/outputs/factor_mix__experts-n2-positioning_check_deepse--positioning_type_deepsee-12c5ae8f9e__target-positioning_check__k5pc__seed42__20260723T031221Z/summary.json)。

## 3. 评测协议

所有完成实验使用同一协议：`seed=42`、5 次 rollout、`temperature=0`、`top_p=1`、`max_model_len=8192`、`max_tokens=512`、`batch_size=64`、`thinking=False`。表中结果为 5 次 rollout 的均值和标准差；Accuracy、macro-F1 和格式有效率均按 `metrics.json` 的 `aggregate` 字段记录。

测试集均为清洗后的固定 final test，与 calibration 分离。样本量：coherence 1,046；positioning_check 603；positioning_type 204；novelty 66；revision 两项各 3,026；Review Utility 中 verifiability 788，其余各 1,000。

## 6. 结果小结与限制

- 5 个已配对任务的 Accuracy 和 macro-F1 均高于基线；最大 Accuracy 增益出现在 positioning_type（+8.33 pp），最大 macro-F1 增益出现在 revision relatedness（+16.51 pp）。
- 除 positioning_type 外，MoDE 的格式有效率下降 0.95--2.29 pp；任务指标提升伴随一定格式稳定性损失。
- novelty 测试集仅 66 条且类别不平衡，波动明显高于其他任务，结论需谨慎。
- 融合权重只在 positioning_check 的 10 条 calibration 上搜索。positioning_type、novelty 和 revision 结果属于迁移评测，不是分别为这些任务重新搜索权重的结果。
- 当前没有 coherence 和 Review Utility 的 MoDE 结果，不能判断融合在这些任务上的效果。
- 现有结果只覆盖一个随机种子、一个专家组合和 `k=5`，尚不能支持专家数、shot 数或随机种子稳健性结论。
