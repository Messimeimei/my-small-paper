# 实验记录

本文件只记录可复核的实验事实：输入、设置、结果、状态和产物。研究动机放在 [研究问题.md](研究问题.md)，选择理由放在 [关键决策.md](关键决策.md)，失败经验放在 [踩坑记录.md](踩坑记录.md)。

证据冲突时按以下顺序取值：结果 JSON / 训练 manifest > 当前数据文件 > README > 周报中的回忆。仅在周报出现、没有产物可核对的数字标为“观察”，不能作为论文正式结果。

状态含义：

- `正式`：设置和结果文件完整，可直接复核。
- `历史`：实验真实发生，但数据或协议已经被替换。
- `部分完成`：只完成了计划中的部分阶段。
- `待复核`：记录或产物存在冲突，暂不用于结论。

## 4. 单专家 LoRA

### 4.1 共用设置

基座为 Qwen3-4B。数据按标签分层固定划分，`split_seed=20260720`；训练 seed 为 42。LoRA 使用 `r=16, alpha=32, dropout=0.05, target_modules=all-linear`；`lr=1e-4`、cosine、3 epochs、BF16、gradient accumulation 16、最大长度 8,192。只对 completion 计算 loss，并关闭 Qwen3 原生 thinking。

### 4.2 训练过程

| ID | 教师数据 | train / val | epoch / step | validation loss | token Acc | 生成 Acc | 运行状态 |
|---|---|---:|---:|---:|---:|---:|---|
| TRAIN-DS-E1 | DeepSeek-v4-pro | 3,266 / 363 | 1 / 205 | 0.6231 | 0.8037 | 未记录 | 继续训练 |
| TRAIN-DS-E2 | DeepSeek-v4-pro | 3,266 / 363 | 2 / 410 | 0.6078 | 0.8067 | 未记录 | 最低 validation loss，保存为最佳 checkpoint |
| TRAIN-DS-E3 | DeepSeek-v4-pro | 3,266 / 363 | 3 / 615 | 0.6128 | 0.8071 | 最终生成验证被中断 | 训练完成；后处理 `KeyboardInterrupt` |
| TRAIN-GLM-E1 | GLM-5.2 | 3,263 / 362 | 1 / 204 | 0.4067 | 0.8689 | 0.8978 | checkpoint 已保存；随后显存不足，未进入第 2 epoch |

不同教师数据的 completion 内容和 validation 子集不同，validation loss 与生成 Acc 不能直接横向比较教师优劣。

## 6. 新实验最小模板

```markdown
### EXP-YYYYMMDD-01：一句话名称

- 研究问题：RQ-xxx
- 唯一变化：相对哪个对照，改了什么
- 固定设置：模型 / 数据版本 / seed / checkpoint / 解码参数
- 结果：主指标 + 格式有效率 + 对照差值
- 状态：正式 / 部分完成 / 待复核
- 产物：结果文件路径
```
