## 共用设置

- 数据：Novelty Alignment（76 条）/ RevUtil-human Actionability（1207 条）

- 输入 prompt 均为 few-shot CoT：按标签类别数（2 或 5）给出对应数量示例，要求模型先输出 `<reasoning>` 再输出 `<score>`；inference 脚本解析 `<score>` 打分并统计指标

- 论文正文仅保留标题 + 摘要 + 介绍 + 结论；prompt 为 **zero-shot**
- `rollout=1`，`max_model_len=16384`，`max_tokens=2048`，`temp=0`，`top_p=1.0`（贪心，便于复现）
- 主指标：10-class Acc / macro-F1（对应论文 Appendix F Table 49 secondary setting + 数据集说明 Acc/F1）
- 随机基线 Acc ≈ 0.10

## 数据说明

### Novelty Alignment（76 条，二分类）

来自论文《Reward Modeling for Scientific Writing Evaluation》。

- 任务：判断针对同一篇论文的两段新颖性评审结论是否一致（novel / not_novel）。每条含一段人工专家 review + 一段第三方大模型（gpt-4o-mini）生成的 review，标签 0/1 表示两者结论是否对齐。
- 标签构建：原始数据无对齐标签，用多个大模型（deepseek-v4-pro、doubao-seed-2-0-pro、glm-5、MiniMax-M3、glm-5.1）分别判断后取多数票；经去无标注 / 去重 / 去多标注后得 76 条，对齐 : 不对齐 ≈ 55 : 21（官方 56 : 20）。
- `score_sets = [0, 1]`，指标为 Acc / F1 / macro-F1。

### RevUtil-human Actionability（1207 条，5 分类）

来自论文《Reward Modeling for Scientific Writing Evaluation》。

- 任务：对一条同行评审意见的「可操作性（actionability）」打 1–5 分——衡量作者读完后能否明确知道要改什么、以及怎么改。
- 标签构建：每条由 3 位博士独立打分；3 人一致为 gold，否则取 2 人一致的分数为 silver（本地用了 gold + silver）。RevUtil-human 原含 actionability / grounding&specificity / helpfulness / verifiability 四个维度，本轮仅评测 actionability。
- `score_sets = [1, 2, 3, 4, 5]`，指标除分类指标外，额外算 Pearson / MSE。

### PeerReview Bench Meta-Reviewer（516 条，10 分类）

来自论文《On the limits and opportunities of AI reviewers》（arXiv:2605.20668）任务二。

- 任务：给定论文正文 + 一条 review item，预测两位专家级联标注压成的 10 类标签（`label_id` 1–10）。
- `score_sets = [1..10]`；本地阉割版仅用 abs-only 论文上下文做 zero-shot 评测。
