# 数据集说明

当前训练数据来自《Reward Modeling for Scientific Writing Evaluation》，包括三个
Related Work Evaluation 二分类任务（`task=rw_gen`）和一个 Review Comment Utility
五分类任务（`task=rev_util`）。模型统一输出
`<reasoning>...</reasoning><score>整数</score>`。

```json

  "completion": [{"role": "assistant", "content": "<reasoning>...</reasoning>\n<score>0或1</score>"}]
}
```

## 3. 数据流水线

```text
origin_data（作者提供的原始训练集）
  -> cleaned_data（去重 / 修正任务描述 / 补稳定 id）
  -> distill_data（教师生成 CoT，按格式和金标签过滤）
  -> lora_data（TRL prompt-completion JSONL）
  -> splits（按 label 分层固定 train / validation）
  -> training/train.py（LoRA SFT）
```

- `coherence`：原始 4,890 条去重为 4,811 条，并修正错误的任务描述；保留原标签边界。
- `positioning_type`、`positioning_check`：原始 prompt 可用，只补稳定 `id`，不改任务内容。
- 蒸馏不额外添加 Zero-shot-CoT；错误答案和格式错误轨迹不进入 LoRA 数据。
- 已生成的 coherence 变体：DeepSeek 3,629 条、GLM 3,625 条、双教师共同正确 3,247 条；当前配置使用 GLM 版本。
- actionability 当前只有 DeepSeek 单教师版本，共 1,788 条；标签范围为 1–5。

各阶段的详细字段与生成方式见 [`cleaned_data/README.md`](train_data/cleaned_data/README.md)、[`distill_data/README.md`](train_data/distill_data/README.md) 和 [`lora_data/README.md`](train_data/lora_data/README.md)。

## 4. 仅用于评测的数据

以下数据不进入当前 LoRA 训练或 train / validation 划分：

| 数据 | 样本数 | 用途 |
|---|---:|---|
| Related Work 三个 test | 1,857 | 测目标 aspect、同任务迁移与能力干扰 |
| RevUtil | 4,788 | 测审稿意见评价迁移 |
| Novelty Alignment | 76 | 测未见新颖性判断任务 |
| Scientific Revision | 6,184 | 测未见科学文本修改评价任务 |
| Meta Reviewer | 779 | 测长上下文下的正确性、重要性和证据判断 |

数据边界：`train_data` 只用于构建训练与验证数据，`test_data` 只用于最终评测；任何测试样本都不得加入蒸馏或固定划分。
