# 1. 科学写作评估数据集

- 论文：《《Reward Modeling for Scientific Writing Evaluation》》
- 数据集介绍：

```text
这篇论文一共提出了四个和科学写作相关的数据集，其中 Related Work 和 RevUtil 是论文用来做两阶段强化学习训练模型的数据，同时也划分了测试集。另外是 novelty 数据和 revision 数据，这两个是直接用训练好的模型进行评测的。
```

- 数据格式说明：
四个数据格式最终都压成同一种 pointwise 打分样本：

```json
{
"task": "rw_gen | rev_util | novelty_eval_updated | revision_eval",
"aspect": "<评估维度>", // 该任务下的具体评估维度
"labels": <金标分数>, // 标签
"score_sets": [<合法分值列表>], // 标签取值范围
"prompt": [
  {"role": "system", "content": "..."},
  {"role": "user", "content": "..."} // 
]
}
```

其中，user prompt 固定骨架：

```text
[QUERY]:      假装在做某类科学写作任务（相关工作/审稿/新颖性/改写）
[CRITERIA]:   评估准则 + 评分量表
[EXAMPLES]:   few-shot（每个合法分数一个示例）
[ANSWER]:     待评估的生成文本（有的任务还会多 ORIGINAL TEXT / INSTRUCTION / CONTEXT）
```

## Related Work（`rw_gen`）

**内容：** 评估 LLM 生成的 related work 段落/引用句是否满足特定写作原则。来源 Sahinuç et al. 2025，并扩展到更多 ML 会议论文。

**规模：** train 8,666 / test 1,857

### 三个 aspect

| aspect | 评什么 | 分值 | Train / Test |
|--------|--------|------|--------------|
| `coherence` | 引用句是否被被引论文 context **蕴含** | 0/1 | 4,890 / 1,048 |
| `positioning_check` | 段落是否（显式/隐式）写出本文相对文献的 **定位/贡献** | 0/1 | 2,822 / 605 |
| `positioning_type` | 是否按指定定位策略组织（如：前文只综述、**末段**再总结贡献） | 0/1 | 954 / 204 |

### 结构差异

- **coherence** 多一段 `[CONTEXT]`（被引论文内容），再评 `[ANSWER]` 里带 citation number 的句子。
- **positioning_*** 主要是段落级：`[QUERY]+[CRITERIA]+[EXAMPLES]+[ANSWER]`，ANSWER 是一段/多段 related work。

**coherence 示意：**

```text
[QUERY]: 根据 context 写与给定 citation number 一致的引用句
[CRITERIA]: context 是否 entail 该引用句 → 0/1
[CONTEXT]: <被引论文片段>
[ANSWER]: "... path-based methods [3], ... require [1] ..."
```

**positioning_check 示意：**

```text
[QUERY]: 写 related work 段落，每段要体现本文定位
[CRITERIA]: 是否提及贡献/定位 → 0/1
[ANSWER]: <一段 related work 文本>
```

## 2. RevUtil（`rev_util`）

**内容：** 评估一条 **审稿意见（review comment）** 对作者是否有用。来源 Sadallah et al. 2025；附录说明人标 + 合成标签合并。Verifiability 拆成两步：先抽 claim，再打可验证程度。

**规模：** train 50,046 / test 4,788

### 五个 aspect

| aspect | 评什么 | 分值 | Train / Test |
|--------|--------|------|--------------|
| `actionability` | 意见是否给出可执行、具体的修改建议 | 1–5 | 10,432 / 1,000 |
| `grounding_specificity` | 是否锚定论文具体位置，并说清问题 | 1–5 | 10,431 / 1,000 |
| `helpfulness` | 综合：是否对改进论文有帮助 | 1–5 | 10,430 / 1,000 |
| `verifiability_extraction` | 文本是否含需论证的 **claim** | 0/1 | 10,430 / 1,000 |
| `verifiability` | （仅含 claim 时）论证是否充分 | 1–5 | 8,323 / 788 |

### 结构（各 aspect 相同骨架，准则不同）

```text
[QUERY]: 写一条审稿意见，且满足某质量要求（可执行/有依据/…）
[CRITERIA]: 该维度定义 + 1–5（或 0–1）量表
[EXAMPLES]: 每个分数一个 ANSWER + EVALUATION 示范
[ANSWER]: <待评审稿意见原文>
```

例如 **actionability**：1=完全不可执行 … 5=明确动作 + 具体做法。  
例如 **verifiability_extraction**：0=纯事实/澄清问句；1=含主观判断、建议、需举证的断言。

## 3. Novelty Alignment（`novelty_eval_updated`）

**内容：** 不直接判论文新不新，而是判 **两段 novelty assessment（人 vs LLM）的最终 verdict 是否一致**（novel / not novel）。来源 Afzal et al.；LLM 侧 verdict 论文里用 GPT 从摘要推断。仅 test。

**规模：** test 76（对齐 56 / 不对齐 20）；`aspect=coherence`；`score_sets=[0,1]`

### 结构

```text
[QUERY]: 写两段新颖性评估，各自隐含 novel/not novel，且应对齐
[CRITERIA]:
  0 = 两段最终结论不一致
  1 = 两段最终结论一致
  （看 final conclusion，不看中间措辞是否相似）
[EXAMPLES]: 对齐 / 不对齐 各一例
[ANSWER]: Assessment 1: ... Assessment 2: ...
```

## 4. Revision（`revision_eval`）

**内容：** 给定原文段落 + 改写指令 + LLM 改写结果，评改写质量。来源 Jourdan et al. 2025。仅 test。

**规模：** test 6,184 = relatedness 3,092 + correctness 3,092；均为 0/1

### 两个 aspect

| aspect | 评什么 | 0 / 1 |
|--------|--------|-------|
| `relatedness` | 改写是否 **按指令** 做了要求的事 | 未按指令 / 按了指令 |
| `correctness` | 改写是否是可替换原文的 **更好版本** | 不能替换 / 应替换 |

### 结构

```text
[QUERY]: 按指令修订科学文本
[CRITERIA]: relatedness 或 correctness 的 0/1 量表
[EXAMPLES]: ORIGINAL TEXT + INSTRUCTION + ANSWER + EVALUATION
[ORIGINAL TEXT]: <原文段落>
[INSTRUCTION]: <改写指令，如 “拆第一句并删第三句”>
[ANSWER]: <模型改写结果>
```

同一 `(原文, 指令, 改写)` 通常会各有一条 relatedness 与 correctness 样本，aspect 与准则不同，`labels` 可不同。

---

## 总览对照

```
科学写作评估数据（SciRM）
├── Seen（训练+测试）→ final_reward_data.json
│   ├── rw_gen      coherence | positioning_check | positioning_type   (0/1)
│   └── rev_util    actionability | grounding | helpfulness | verif*  (0/1 或 1–5)
└── Unseen（仅测试）
    ├── prompted_novelty_data.json   → 两段 assessment 是否同 verdict (0/1)
    └── prompted_revision_data.json  → relatedness / correctness (0/1)
```

**共同点：** 都是「准则 + few-shot + 待评文本 → 输出 reasoning + score」。  
**差别：** Related Work 评相关工作文本；RevUtil 评审稿意见；Novelty 评两段结论对齐；Revision 多 `ORIGINAL TEXT`/`INSTRUCTION`，评改写是否听话、是否更好。

# 2. PeerReview Bench 数据集

* 论文：《On the limits and opportunities of AI reviewers: Reviewing the reviews of Nature-family papers with 45 expert scientists》(arXiv:2605.20668)
* 来源：[https://huggingface.co/datasets/prometheus-eval/peerreview-bench](https://huggingface.co/datasets/prometheus-eval/peerreview-bench)
* 数据说明：
这个数据集用来评测模型的科学文献审稿能力。数据集中的几个文件介绍如下：

| 文件                  | 干什么                                                |
| ------------------- | -------------------------------------------------- |
| `expert_annotation` | 专家已经标好的「哪些人工意见是好意见」→ 用来算 **Recall**                |
| `meta_reviewer`     | 另一条支线：测「模型能不能像专家一样给某条意见打分」；主评测的 Precision 也要用类似打分器 |
| `reviewer`          | 给模型看的论文正文（78篇论文正文）                                 |
| `submitted_papers`  | 论文附件（代码等），需要时再取                                    |
| `similarity_check`  | 判断两条意见是不是在说同一件事                                    |

## 任务介绍

| 任务                                           | 数据                                                    | 评测对象            | 指标            |
| -------------------------------------------- | ----------------------------------------------------- | --------------- | ------------- |
| 1. 输入论文信息，模型给出审稿意见，由 meta-reviewer 打分，最后计算指标 | `reviewer` + `submitted_papers` + `expert_annotation` | 模型生成的审稿意见       | P / R / F1    |
| 2. 输入审稿意见，模型给出对不对、重不重要、证据够不够的级联判断结果          | `meta_reviewer`（± `submitted_papers`）                 | 模型给出的判断标签（1-10） | Acc / F1      |
| 3. 输入两条审稿意见，模型给出相似度判断（0-1）                   | `similarity_check`                                    | 模型给出的审稿意见相似度判断  | Acc（/ 细粒度 F1） |

## meta_reviewer 数据集

这个任务是：给定一篇论文与一条审稿意见（review_item），让模型像 meta-reviewer 一样，按级联规则标注该意见的 **correctness → significance → evidence**，并预测两位专家是否会在各维度上达成一致。标签被压成 10 类（`label_id` 1–10 / `label` 字符串）。共 908 条，仅包含同时有 primary 与 secondary 两位专家标注的样本。

10 类标签含义：

| ID  | Label                                    | 含义                                                  |
| --- | ---------------------------------------- | --------------------------------------------------- |
| 1   | correct_significant_sufficient           | 两位：Correct + Significant + Sufficient               |
| 2   | correct_significant_insufficient         | 两位：Correct + Significant + Requires More            |
| 3   | correct_significant_disagree_on_evidence | 两位：Correct + Significant，但 evidence 不一致             |
| 4   | correct_marginal_sufficient              | 两位：Correct + Marginally Significant + Sufficient    |
| 5   | correct_marginal_insufficient            | 两位：Correct + Marginally Significant + Requires More |
| 6   | correct_marginal_disagree_on_evidence    | 两位：Correct + Marginally Significant，evidence 不一致    |
| 7   | correct_not_significant                  | 两位：Correct + Not Significant                        |
| 8   | correct_disagree_on_significance         | 两位：Correct，但 significance 不一致                       |
| 9   | incorrect                                | 两位：Not Correct                                      |
| 10  | disagree_on_correctness                  | 两位在 correctness 上不一致                                |

## expert_annotation 数据集

这个任务是专家对每条审稿意见做级联标注：先判 correctness（Correct / Not Correct）；仅当 Correct 时再判 significance（Significant / Marginally Significant / Not Significant）；仅当至少 Marginally Significant 时再判 evidence（Sufficient / Requires More）。每条记录对应 (paper, reviewer, review_item, annotator_source)，primary/secondary 各一行。共 3881 条（primary 2960 / secondary 921）。

## similarity_check 数据集

这个任务是判断同一篇论文下两条审稿意见是否在讨论同一问题：给出 `binary_label`（similar / not_similar）以及更细的 `finegrained_label`（4 类）。共 164 对（similar 70 / not_similar 94；AI-AI 85 / AI-Human 79）。

## reviewer / submitted_papers

- **reviewer**（78 条）：每篇论文一行，含 `paper_content`、`file_refs`、`rubric`，用于评估「AI 审稿生成」；生成结果可与 `expert_annotation` 中的 ground-truth 审稿意见对比。
- **submitted_papers**（22351 条，约 3.4GB）：按 SHA256 去重的论文附属文件二进制存储；通过其他 config 的 `file_refs.content_hash` 关联读取。

本地加载示例：

```python
from datasets import load_dataset
base = 'data/peerreview-bench'
meta = load_dataset(base, 'meta_reviewer', split='eval')       # 908
expert = load_dataset(base, 'expert_annotation', split='eval') # 3881
sim = load_dataset(base, 'similarity_check', split='eval')     # 164
papers = load_dataset(base, 'reviewer', split='eval')          # 78
files = load_dataset(base, 'submitted_papers', split='eval')   # 22351
```
