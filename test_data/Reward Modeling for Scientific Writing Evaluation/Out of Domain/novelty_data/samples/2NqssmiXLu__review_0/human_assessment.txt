This paper proposes a method for finetuning a code-generating large language model (LLM) to generate verification annotations for Rust code, specifically targeting the Verus verifier. The approach addresses the challenge of limited data resources by bootstrapping the finetuning process: it first generates proof specifications using GPT-4o, filters these for quality, then uses expert-crafted prompts to generate proof annotations for a subset, and finally iteratively finetunes the model using both correct and incorrect proofs. The most significant contribution lies in tackling the novel and impactful task of verifying Rust code with LLMs—a task for which there is little prior work or available data, and the authors are commended for their data collection efforts. However, while the task itself is new, the technical approach—bootstrapping finetuning by interleaving it with search—closely resembles the established "expert iteration" framework (Polu et al., 2022), and the paper may overstate the novelty of its "SAFE" method. In particular, the initial "SAFE_0" stage appears to be standard synthetic data generation and filtering using GPT-4o, rather than a fundamentally new technique. The truly novel aspects of the work are the specific application to Rust verification, the use of finetuning on repair tasks to improve generation, and the metric for filtering specification samples. The paper would benefit from more clearly distinguishing its method from prior expert iteration approaches and focusing its novelty claims on these genuinely new contributions.
