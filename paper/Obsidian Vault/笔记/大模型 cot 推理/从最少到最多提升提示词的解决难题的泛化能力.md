# 核心解决问题

cot 在解决给出示例简单但题目更困难的时候，效果不行，即 cot 在难题的泛化性上较差。因此文章提出了 least-to-most 的提示词方法，帮助模型解决更复杂的推理问题

# 核心做法

通过多次调用模型，让模型自己多次拆解问题，并且每次把子问题和答案加到 prompt 中。在第一次的 prompt 中给出与要解决的问题无关的拆解问题的示例和要解决的问题，然后由模型拆解要解决的问题得到子问题和答案；第二次 prompt 中包含了拆解示例+（子问题，答案）+最终问题。如此循环直到模型给出最终问题的答案

# 不足与研究方向

1. least-to-most 方法存在领域局限性，一个领域的分解无法扩展到其他领域，不具有通用性
    
2. 本质上是单向的教导模型，如果开头分解错误，会影响后续结果（加入ReAct，引入反思过程，引导模型纠正）
    
3. 计算和时间成本大，无法运用到实时场景（蒸馏到小模型，分解过程一次性给到小模型；或者压缩推理过程，紧凑型推理）
    
4. 依赖模型尺寸，小模型效果不行
