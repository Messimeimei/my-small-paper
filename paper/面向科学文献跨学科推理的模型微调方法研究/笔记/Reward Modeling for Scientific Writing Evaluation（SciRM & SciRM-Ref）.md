## 一、研究背景与动机

1. **行业痛点**
    
    科研文本生成（相关工作撰写、论文审稿意见生成、稿件修订）已广泛使用大模型，但**科学类文本缺少可靠自动化评估方案**。人工评审成本极高，通用 LLM 充当评审器存在明显缺陷：无法深度理解领域专业知识、难以严格遵循多维度、动态自定义评分细则，跨任务泛化能力差。
2. **现有缺陷**
    
    - 通用奖励模型、LLM-as-judge 仅针对通用基准优化，适配固定打分标准，无法处理科学写作多维度、领域稀疏知识的评估需求；
    - 针对单一科研任务微调评估模型成本极高，低资源场景下无法落地；
    - 现有评估方案多输出单一总分，缺少细粒度多维度打分，评估结果可解释性差；
    - 主流奖励模型采用成对偏好训练，无法基于显式任务细则独立完成分项评估，遇到训练未见过的评估维度、全新科研任务性能大幅衰减。
    
3. **研究目标**
    
    提出低成本开源科学写作专用奖励模型 **SciRM、SciRM-Ref**，设计两阶段 GRPO 强化学习训练框架，支持基于显式评估细则（评估标准 + 打分样例）完成多维度细粒度打分，实现跨任务、未见过评估维度 / 未见过科研任务的强泛化，一套模型适配全部科学写作评估场景，无需针对单任务重新微调。

## 二、核心创新点（四大核心贡献）

1. **C1：领域专用开源奖励模型**
    
    面向科学写作评估设计轻量化奖励模型 SciRM（单阶段训练）、SciRM-Ref（双阶段推理自校正训练），基于开源 7B 底座，无需闭源大模型，低成本适配科研场景。
2. **C2：两阶段 GRPO 强化学习优化流水线**
    
    第一阶段优化模型对科学评估偏好的拟合能力，让模型严格遵循输入的评估细则、打分标准；第二阶段引入自反思校正机制，提升复杂推理能力，模型可自主复盘自身推理过程、修正初始打分错误，适配高推理难度的科学评估任务。
3. **C3：多维度分项评估范式**
    
    摒弃单一总分输出，对科学文本开展多维度独立打分，每个维度附带完整推理过程，大幅提升评估结果可靠性与可解释性，输出固定结构化标签`<reasoning></reasoning><score></score>`便于自动解析奖励。
4. **C4：多任务联合训练增强泛化**
    
    融合多类科学写作任务数据联合训练，模型学习通用评估逻辑，面对训练未见过的评估维度、全新科学写作任务仍保持稳定性能，解决单任务过拟合、跨任务性能断崖下降问题。

## 三、数据集构建与数据处理

### 1. 数据来源

融合两套公开科学评估数据集构建混合训练集，总样本 65357 条，划分训练集 58712 条、测试集 6645 条：

- 相关工作评估数据集：包含连贯性、段落定位类型、定位一致性 3 个二元评估维度，合计 10523 条；
- RevUtil 审稿效用数据集：包含可操作性、依据充分性、可验证性、有用性 5 个 1–5 分连续打分维度，合计 54834 条。
    
    额外引入两类**测试集未参与训练**，用于泛化验证：新颖度对齐评估数据集、科学稿件修订评估数据集。

### 2. 数据预处理流程

1. 扩充原仅覆盖 NLP 领域的相关工作数据集，新增机器学习顶会论文样本，统一原文预处理逻辑；
2. 合并 RevUtil 人工标注与合成标注样本，拆分可验证性为 “可验证提取（二元）”“可验证程度（1–5 分）” 两个子维度；
3. 统一所有样本输入格式：`任务查询q + 评估细则c + 打分示例e`，标准化 prompt 模板；
4. 划分分层训练 / 测试集，保证各评估维度样本分布均衡。

### 3. 覆盖全部评估任务维度

训练阶段覆盖 4 大类科学写作评估任务：相关工作评估、审稿意见效用评估；

泛化测试新增两类零样本任务：论文新颖度对齐评估、稿件修订质量评估；

每个任务包含独立细分评估维度、专属打分规则（二元 0/1、1–5 五分制）。

## 四、模型训练与奖励设计

### 1. 基础底座与训练框架

基座选用 Qwen2.5-7B，采用 LoRA 轻量化微调（LoRA 秩 = 64），基于 Unsloth、vLLM 加速，单卡 A100 80GB 完成训练，全程约 3 天；使用 GRPO 强化学习算法完成两阶段优化。

### 2. 第一阶段训练（SciRM 基础模型）

**目标**：学习遵循评估细则、规范输出格式、正确打分。

1. 奖励函数设计：区分格式错误、分值越界、分值正确但答案错误、分值完全正确四类情况，分配梯度奖励；
2. 引入长度惩罚项：惩罚过短 / 过长输出，缓解奖励投机、生成截断、无意义超长文本问题；
3. 最终单样本总奖励 = 打分匹配奖励 − 文本长度惩罚。

### 3. 第二阶段自校正训练（SciRM-Ref）

**目标**：提升深度推理、自我纠错能力。

1. 流程：模型先输出初始推理与初始分数，擦除分数部分，追加反思提示词，让模型重新复盘评估标准、输出修正后最终分数；
2. 分段奖励机制：重点奖励 “初始打分错误、反思后修正正确” 样本，惩罚 “初始正确反思后改错” 样本，强制模型学会校验自身逻辑；
3. 消融实验证明：推理文本是模型高精度打分的核心，移除推理模块后所有任务性能下降，高推理难度任务（新颖度对齐）降幅最大（-0.25）。

## 五、基线对比模型

1. 开源同尺寸通用底座：DS-R1-Qwen7B、Granite3.3-8B、Llama3.1-8B、Qwen3-8B；
2. 闭源强基线：GPT-5.2、o3-mini；
3. 通用评估专用模型：Prometheus、Selene；
4. 通用奖励模型基线：DeepSeek-GRM、Skywork-Critic。

## 六、实验结果（SciRM / SciRM-Ref vs 各类基线）

实验分为三大验证场景：训练见过任务、训练未见过评估维度、训练完全未见过全新任务，指标采用准确率、F1 分数。

1. **训练见过任务（相关工作、审稿效用评估）**
    
    SciRM、SciRM-Ref 整体准确率大幅超越全部开源基线，多数维度接近或持平 GPT-5.2 闭源模型；审稿效用全任务平均精度 0.69–0.71，显著高于通用奖励模型（0.44–0.57）；相关工作定位一致性维度接近满分。
2. **掩码消融实验（未见过评估维度）**
    
    移除训练集中 Actionability、Grounding 维度得到 SciRM-MASKED，在未见过维度上性能小幅下降，但仍全面优于绝大多数开源基线，证明模型学习通用评估逻辑，未过拟合训练维度。
3. **零样本全新任务（泛化能力核心验证）**
    
    - 新颖度对齐评估（高推理难度）：SciRM-Ref 准确率 0.74，远超 SciRM（0.61），仅次于闭源 o3-mini（0.78），证明第二阶段反思训练显著提升复杂推理；
    - 稿件修订评估：SciRM 系列模型与最优基线持平，在相关性、正确性维度稳定领先通用奖励模型；
    
4. **全任务平均综合性能**
    
    SciRM-Ref 四项任务平均准确率 0.76，为所有开源模型最优，仅低于闭源 o3-mini（0.75）、GPT-5.2（0.74）；SciRM 平均 0.73，大幅领先所有通用开源奖励模型。
5. **定性推理对比结论**
    
    基线模型推理存在三大缺陷：脱离评估细则主观臆断、权重错配评估维度、遗漏文本关键细节；SciRM 严格基于给定标准推理，SciRM-Ref 可自主修正推理偏差，推理逻辑与最终打分完全自洽。
6. **现存短板**
    
    实验仅使用 7B 小尺寸基座，更大规模模型理论上可捕捉更细微领域评估特征；科学评估标注数据集稀缺，多数维度仅支持二元打分，缺少大量细粒度连续打分标注数据。

## 七、核心结论

1. 针对科学写作场景定制两阶段奖励建模方案，能够解决通用 LLM / 通用奖励模型领域知识不足、无法适配动态评估细则、跨任务泛化差的痛点；
2. 仅第一阶段训练的 SciRM 即可在训练见过任务上实现 SOTA，增加第二阶段反思自校正后 SciRM-Ref 在强推理零样本任务上获得显著提升；
3. 多任务联合训练使模型习得可迁移的通用评估逻辑，无需针对新科学写作任务重新微调，大幅降低科研场景自动化评估落地成本；
4. 多维度分项 + 可解释推理文本的输出范式，相比单一总分，评估结果更可靠、更贴合人工审稿人的评判逻辑。

## 八、未来研究方向

1. 利用本文训练好的 SciRM/SciRM-Ref 作为奖励信号，通过强化学习优化科学文本生成模型（相关工作、审稿、论文修订）；
2. 扩充高质量多粒度科学写作评估标注数据集，丰富 1–5 分细粒度打分样本；
3. 扩大模型基座参数规模，验证大尺寸模型在科学评估任务上的性能上限；
4. 拓展多模态科学文本评估（论文图表、公式、实验结果），构建图文联合评估奖励模型。

## 九、研究贡献总结

1. 提出两套开源科学写作专用奖励模型 SciRM、SciRM-Ref，创新两阶段 GRPO 训练框架，引入反思自校正机制提升复杂领域推理打分能力；
2. 构建融合多类科学写作任务的混合标注数据集，公开训练代码与全部数据，为领域自动化评估提供开源基线；
3. 验证多维度分项评估、基于显式细则条件化奖励建模的有效性，一套模型支持已知 / 未知评估维度、全新零样本科学写作评估任务；
4. 系统对比通用 LLM、通用奖励模型、闭源商用模型，定量证明领域专用奖励模型在科学评估任务上的显著优势，为科研文本自动化评测提供低成本、可复用完整训练流水线。
