## 1. 选题方向

初步选定是，解决小模型的推理优化问题。大的方向是，大模型推理有成本问题，以及在特定领域的推理效果不佳。于是引出了用小模型降低成本问题，同时其推理效果要达到大模型的一定程度，甚至超过已有的方法。
同时具体到某一个领域上，去验证这一套思路，即在科学文献领域。于是需要确定数据集（问题）、模型（方法），和自己的创新点，并且做实验去验证自己的模型可以提升在科学文献领域特定数据集上的推理准确率和可信度，并证明成本相对较低。
**如何让小模型在进行科学文献领域适配时，不牺牲原有推理能力，并在专业推理任务上进一步提升？**
## 2. 数据集
初步思路是，可以在已有的科学文献问题上看看加入推理后的准率率，同时再看看其他模型解决不了的复杂一点的任务的准确率。

如何提升小模型在复杂专业文本上的推理能力，同时保持较低推理成本？

| 用途         | 数据集                      | 作用                 |     |
| ---------- | ------------------------ | ------------------ | --- |
| **领域训练**   | SciRIFF                  | 科学文献领域适配（LoRA/SFT） |     |
| **领域推理测试** | SciFact、SciAssess、QASPER | 验证科学文献推理能力         |     |
| **通用推理测试** | GSM8K、BBH                | 验证推理能力没有因领域适配而退化   |     |
### 不一定是科学文献垂域数据集

- **数据集 A：SciRIFF (Scientific Review and Information Extraction)**
    
    - **它是什么：** 目前最火的、专门用于科学文献指令微调的大型高质数据集。包含论文摘要抽取、因果关系识别、学术观点验证等数万条高质量科学文本对。
        
    - **怎么用：** 它是你 **Domain-LoRA** 最完美的“精神食粮”。
        
- **数据集 B：MSM (Multi-Disciplinary Scholarly Methodology)**
    
    - **它是什么：** 涵盖计算机、生物、材料等多个学科的学术论文方法论数据集。
        
    - **怎么用：** 用它来测试你的模型在学了高密度知识后，能不能理清复杂的、跨学科的**学术方法论推理**。、
    https://github.com/allenai/scifact

### 也可以是通用领域数据集和其他领域的如医学啥的
为了证明你的方法保持了小模型的推理能力，你必须在微调后，去跑大模型界公认的推理大考：
GSM8K / MATH： 测试小模型的数学与多步逻辑推导能力。
BBH (BIG-bench Hard)： 包含大量反事实推理、逻辑网格推理的任务。
